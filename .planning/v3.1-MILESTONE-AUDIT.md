## Cross-Phase Integration

**Integration checker result:** 14/14 connections wired | 0 orphaned exports | 0 broken connections

### Verified Wiring Chains

- **Lifecycle chain:** AgentSidebar.StartButton → App.tsx handlers → instanceRoutes (start/stop/restart/force-kill) → TmuxSessionManager → InstanceTracker → DatabaseConnection → useActiveInstances → InstanceTabBar/TerminalView
- **Budget chain:** TokenUsageView (window selector + burn rate cards + budget editor) → burn-rate/budget-config API endpoints → DatabaseConnection → useBudgetAlerts hook → App.tsx History nav badge (amber/red)
- **Analytics chain:** SessionUsageReader JSONL scanner → token_usage_by_model table → getModelComparison() → historyRoutes → ModelComparisonView (bar chart + insight) → TokenUsageView tabs
- **Export chain:** TokenUsageView handleExport → /api/history/token-usage/export → getTokenUsageForExport → CSV blob download → toast notification
- **Filter chain (Phase 26):** agentFilter input in shared TokenUsageView header → ModelComparisonView prop → useEffect re-fetch on change → API agentId param → DB WHERE clause

### Cross-Phase Notes

- Phase 21 → Phase 22: Both poll independently (instances 5s, budget alerts 30s) — no direct data coupling, correct by architecture
- Phase 22 → Phase 23: Both tables populated in same SessionUsageReader scan loop — intentionally independent surfaces
- CSV export returns full unfiltered dataset regardless of agentFilter (intentional per Phase 23 spec)
