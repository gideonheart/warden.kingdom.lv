## Tech Debt

### Phase 24: Session Recording & Replay (2 items)

1. **Recording state desync on session_ended** — `useRecordingState` hook doesn't listen for `terminal:exit` Socket.IO event. When PTY dies naturally, server auto-stops recording correctly, but client `isRecording` stays true until the stopped/error overlay covers the header. **Fix:** Add `terminal:exit` listener in hook, or let `App.tsx` `handleSessionExit` clear recording state via callback.

2. **Unused `__dirname` variable** — `RecordingCaptureService.ts` line 6: `const __dirname` declared but only used for `path.dirname` setup. Info-level, no functional impact, typecheck passes.

### Phase 23: Token Analytics & Export (3 items)

3. **Verification doc inconsistency** — `23-VERIFICATION.md` frontmatter says "8/8 must-haves" (post-fix) but body text still says "7/8 plan truths verified" (pre-fix). Cosmetic — should be refreshed for consistency.

4. **Duplicated `formatAgentId()`** — Helper exists in both `TokenUsageView.tsx` and `ModelComparisonView.tsx`. Deliberate per 23-02 decision to avoid cross-component coupling.

5. **Export dumps full dataset** — Export CSV returns full unfiltered data regardless of active `agentFilter`. Matches TOKN-14 spec ("full token usage dataset") but may surprise users who have filtered the view.

**Total: 5 items across 2 phases — 0 blockers, all non-critical**
