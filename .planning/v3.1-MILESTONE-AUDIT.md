## Tech Debt

### Phase 21: Agent Lifecycle Controls
- **Info:** Unused `import { openSync, closeSync } from 'fs'` in `instanceRoutes.ts` (line 3) — dead code carry-over, no functional impact

### Phase 23: Token Analytics & Export
- **UX:** `formatAgentId()` duplicated in `TokenUsageView.tsx` and `ModelComparisonView.tsx` — deliberate decision per 23-02 SUMMARY to avoid cross-component coupling
- **UX:** Export CSV always dumps full unfiltered dataset regardless of active `agentFilter` state — matches TOKN-14 spec ("full token usage dataset") but may surprise users who have filtered the view
- **Cosmetic:** `calculateDateFrom('24h')` in ModelComparisonView returns today's calendar date, not rolling 24 hours — consistent with burn rate "today" semantics but "24h" label creates expectation mismatch

**Total: 4 items across 2 phases (all non-blocking)**

## Database & Route Health

- **Migrations:** All 5 migration blocks run sequentially in `DatabaseConnection.runMigrations()` — base schema → ALTER columns → budget_config → token_usage_by_model. Correct ordering confirmed.
- **Route ordering:** `/budget-config/status` registered before `/:agentId` — no param shadowing. `/token-usage/export` correctly matches before `/token-usage` base path. No conflicts.
- **Polling intervals:** `useActiveInstances` (5s), `useBudgetAlerts` (30s), burn-rate/model-comparison (on-demand). No conflicting poll rates.
- **Agent ID namespaces:** `instances` table uses tmux session prefixes (`warden`, `scout`), token tables use project directory names (`home-forge-warden-kingdom-lv`). Two separate identifier spaces, never joined — correct by design.
