**2. Create `OpenClawSessionReader`** at `src/server/services/OpenClawSessionReader.ts`:

Single service with two responsibilities (both read-only lookups from external OpenClaw data sources):

**A) Agent registry — working directories (cached, file-based):**
- Read `/home/forge/.openclaw/workspace/skills/gsd-code-skill/config/agent-registry.json`
- Follow `OpenClawConfigReader` caching pattern (30s TTL, return stale cache on error)
- Method: `getWorkingDirectories(): Promise<Map<string, string>>` — returns `agent_id → working_directory`
- ENOENT → return empty Map silently (skill not installed)
- Registry interface: `{ agents: Array<{ agent_id: string; working_directory: string; [key: string]: unknown }> }`

**B) Context fill — openclaw CLI (cached 30s):**
- Shell out: `execFile('openclaw', ['sessions', '--all-agents', '--json'])`
- This returns `{ sessions: Array<{ key, agentId, totalTokens, contextTokens, model, ageMs, ... }> }`
- Cache the parsed result for 30s (same TTL as registry)
- Method: `getContextFills(): Promise<Map<string, ContextFill>>` where `ContextFill` = `{ totalTokens: number | null; contextTokens: number; fillPercentage: number | null; model: string | null }`
- Filter sessions to only those with key ending in `:main`
- Calculate: `fillPercentage = totalTokens != null && contextTokens > 0 ? Math.round((totalTokens / contextTokens) * 100) : null`
- If CLI fails (not installed, timeout, bad JSON): return empty Map, log warning once
- Use `execFile` from `child_process` (not `exec`) — same pattern as `TmuxSessionManager`

Export singleton: `export const openClawSessionReader = new OpenClawSessionReader();`

- Return enriched agents. No new endpoint needed — client hook already polls `/api/agents` every 30s.
