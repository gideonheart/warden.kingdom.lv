--- Describe block 1: "useSessionSelection — manual selection persistence" ---

Test cases (use renderHook + act):

1. "auto-selects first session on initial load completion"
   - Render with activeInstances=[], isLoading=true, initialSessionName=null
   - result.current.selectedSessionName should be null
   - act: rerender with activeInstances=[makeInstance('session-a')], isLoading=false
   - result.current.selectedSessionName should be 'session-a'

2. "selectSession() takes effect immediately and persists through the next poll"
   - Render with activeInstances=[makeInstance('session-a'), makeInstance('session-b', {id:2})],
     isLoading=false, initialSessionName='session-a'
   - act: call result.current.selectSession('session-b')
   - selectedSessionName should be 'session-b'
   - act: rerender with same activeInstances (simulating next poll, identical data)
   - selectedSessionName should still be 'session-b'

3. "selectSession() resets the miss counter so the session is not immediately evicted"
   - Render with activeInstances=[makeInstance('session-a'), makeInstance('session-b', {id:2})],
     isLoading=false, initialSessionName='session-a'
   - act: call result.current.selectSession('session-b')
   - act: rerender with activeInstances where session-b is absent (simulate first miss)
   - selectedSessionName should still be 'session-b' (hysteresis: 1 miss tolerated)
   - act: rerender again with session-b still absent (second consecutive miss)
   - selectedSessionName should fall back to 'session-a' (first available)

4. "clearSelection() sets selectedSessionName to null"
   - Render with activeInstances=[makeInstance('session-a')], isLoading=false,
     initialSessionName='session-a'
   - act: call result.current.clearSelection()
   - selectedSessionName should be null

--- Describe block 2: "useSessionSelection — polling interactions" ---

Test cases:

5. "tolerates one missed poll (hysteresis) before changing selection"
   - Render with activeInstances=[makeInstance('session-a')], isLoading=false,
     initialSessionName='session-a'
   - act: rerender with activeInstances=[] (session-a disappears — first miss)
   - selectedSessionName should still be 'session-a'

6. "falls back after two consecutive missed polls"
   - Render with activeInstances=[makeInstance('session-a'), makeInstance('session-b', {id:2})],
     isLoading=false, initialSessionName='session-a'
   - act: rerender with activeInstances=[makeInstance('session-b', {id:2})] (first miss for session-a)
   - selectedSessionName should still be 'session-a'
   - act: rerender again with activeInstances=[makeInstance('session-b', {id:2})] (second miss)
   - selectedSessionName should now be 'session-b'

7. "does not change selection while isLoading is true"
   - Render with activeInstances=[makeInstance('session-a')], isLoading=false,
     initialSessionName='session-a'
   - act: rerender with activeInstances=[], isLoading=true
   - selectedSessionName should still be 'session-a'

8. "miss counter resets when session reappears after one miss"
   - Render with activeInstances=[makeInstance('session-a')], isLoading=false,
     initialSessionName='session-a'
   - act: rerender with activeInstances=[] (first miss — still 'session-a')
   - act: rerender with activeInstances=[makeInstance('session-a')] (session returns)
   - selectedSessionName should be 'session-a'
   - act: rerender with activeInstances=[] again (new first miss — should tolerate)
   - selectedSessionName should still be 'session-a' (confirms counter was reset)

- Wrap all state-changing calls (rerender, selectSession, clearSelection) in act().
- Do NOT use fake timers — this hook has no setTimeout/setInterval, just useEffect.
- The hook uses useEffect triggered by prop changes; act() ensures effects flush synchronously
  in the test environment.
- Follow the @testing-library/react pattern already established in useTerminalSocket.test.ts.
