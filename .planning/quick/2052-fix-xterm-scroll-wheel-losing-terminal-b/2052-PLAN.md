**1. Register xterm.js parser handlers to suppress mouse tracking modes (after `terminal.open()`, around line 433):**

Register handlers for the four mouse tracking enable sequences:
- `\x1b[?1000h` — Basic mouse tracking
- `\x1b[?1002h` — Button-event tracking
- `\x1b[?1003h` — Any-event tracking
- `\x1b[?1006h` — SGR extended mouse mode

Use `terminal.parser.registerCsiHandler({final: 'h'}, params => { ... })`. The handler receives the parsed parameter array. Check if the parameter is a "private" DEC sequence (the `?` prefix) by checking `params.length >= 1` and whether the param value matches 1000, 1002, 1003, or 1006. The CSI `?` prefix is signaled via the parameter being in the "private" range — in xterm.js v5, private params have bit 30 set (value OR'd with 0x3f000000), but for DEC private params accessed via `IParams`, use a Set check approach:

```typescript
const MOUSE_TRACKING_MODES = new Set([1000, 1002, 1003, 1006]);
// Register a CSI handler for 'h' (Set Mode) with the '?' prefix
const disposeMouseHandler = terminal.parser.registerCsiHandler(
  { final: 'h', prefix: '?' },
  (params) => {
    // If ANY param is a mouse tracking mode, suppress the entire sequence
    for (let i = 0; i < params.length; i++) {
      if (MOUSE_TRACKING_MODES.has(params[i])) {
        return true; // true = handled, suppress default processing
      }
    }
    return false; // false = not handled, let xterm.js process normally
  }
);
```

Store `disposeMouseHandler` (it's an `IDisposable`) and call `.dispose()` in the cleanup function.
