Perform a thorough code review of commit 68cbdf1 by examining these specific edge cases:

1. **Error path regression**: When a fetch fails (catch block), `setError(message)` is called but the refs are NOT updated. Verify this is correct behavior (it is -- error should not change cached data, and next success correctly compares against last known good state).

Check whether Express/server code returns deterministic field order. This is a theoretical concern -- V8's JSON.stringify preserves insertion order, and Express `res.json()` serializes then the client `response.json()` deserializes with consistent V8 ordering. Confirm this is a non-issue for this codebase.

4. **Comparison with sibling hooks**: Verify pattern consistency:
   - `useActiveInstances`: Uses `computeInstanceSignature()` which extracts only rendering-relevant fields (id, tmuxSessionName, status) and sorts. MORE selective but also MORE complex.
