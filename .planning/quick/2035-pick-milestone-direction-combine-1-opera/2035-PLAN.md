**Header:** Milestone name `v3.0 Operator Awareness & Terminal Power Tools` (or similar — combine both themes into a cohesive name). Date: 2026-03-03. Status: Scoping.

**Motivation section:** Why these two areas combine well:
- Both improve the operator's ability to monitor and interact with agent sessions without leaving the Warden dashboard
- Operator Awareness is about passive monitoring (alerts come TO you), Terminal Power Tools is about active investigation (you go TO the data)
- Together they make Warden a complete operator workstation, not just a viewer

**Feature Area 1 — Operator Awareness:** Detail these sub-features with priority (P1/P2/P3):

P1 (Phase 1 candidates):
- **Permission prompt detection**: Detect when Claude Code is waiting for user permission (regex on terminal output or PTY tap). Show a visual alert badge on the session tab and optionally a toast/notification. This is the highest-value feature — permission prompts block agent progress silently.
- **Context window pressure alert**: Surface the context pressure percentage already extracted by `extractContextPressure()` in `useAgentLiveStatus.ts` (Phase 14). Currently only shown in GSD Agents tab — needs to be visible in the main terminal view as a badge/indicator when pressure exceeds a threshold (e.g., 70% warning, 90% critical).

P2 (Phase 2 candidates):
- **Agent state banner in terminal view**: Show the detected agent state (idle/working/error/permission_prompt) as a subtle overlay or status bar within the terminal view itself, not just in the GSD Agents tab.
- **Notification sound/browser notification**: Optional browser Notification API or audio cue when permission prompt detected or context pressure hits critical.

P3 (Deferred/stretch):
- **Telegram integration for alerts**: Forward permission prompts or context pressure warnings to the operator's Telegram (via existing Gateway topic mapping).

**Feature Area 2 — Terminal Power Tools:** Detail these sub-features with priority:

P1 (Phase 1 candidates):
- **Terminal text search (Ctrl+F)**: Implement xterm.js SearchAddon to find text in the terminal buffer. Show a search bar overlay (input + prev/next/close) triggered by Ctrl+F or a toolbar button. This is the most-requested terminal improvement — operators currently scroll manually through massive buffers.
- **Keyboard navigation shortcuts**: Document and implement keyboard shortcuts for core actions: switch tabs (Ctrl+1-9 or Alt+Left/Right), focus terminal (Escape), open search (Ctrl+F), toggle sidebar (Ctrl+B or similar).

P2 (Phase 2 candidates):
- **Search highlight persistence**: Keep search highlights visible while navigating results. Show match count (e.g., "3 of 17 matches").
- **Quick-scroll markers**: Visual markers in a scrollbar gutter showing where search matches or permission prompts appear in the buffer.

P3 (Deferred/stretch):
- **Terminal bookmarks**: Let operator mark positions in the scrollback buffer for later reference.
- **Regex search**: Support regex patterns in terminal search.

**Phase breakdown:** Propose 3-4 phases:

Phase 19 — "Permission Alerts & Terminal Search" (P1 features, ship first):
  - Permission prompt detection + tab badge alert
  - Context pressure badge in terminal view header
  - xterm.js SearchAddon integration (Ctrl+F search bar)
  - Basic keyboard shortcuts (tab switching, focus, search)
  Acceptance: Operator sees permission badge within 5s of prompt appearing. Ctrl+F opens search overlay and finds text in buffer. Context pressure shows in terminal header when > 70%.

Phase 20 — "Enhanced Alerts & Search Polish" (P2 features):
  - Agent state banner/status bar in terminal view
  - Browser notifications for permission prompts (opt-in)
  - Search match count and highlight persistence
  - Scrollbar gutter markers for search results
  Acceptance: State banner visible in terminal view. Browser notification fires on permission prompt. Search shows "N of M matches" with prev/next navigation.

Phase 21 — "Stretch Goals" (P3, only if time/value):
  - Telegram alert forwarding
  - Terminal bookmarks
  - Regex search
  Acceptance: Defined per feature if pursued.

**Technical notes section:**
- `xterm-addon-search` is the official xterm.js search addon — check npm for compatibility with xterm.js 5
- Permission prompt detection: look for patterns like "Do you want to proceed?" / "Allow" / Claude Code's permission prompt format in PTY output. May reuse the PTY tap mechanism from Phase 11 (TerminalStreamService event capture with setImmediate).
- Context pressure already parsed by `extractContextPressure()` — needs to be surfaced via Socket.IO or REST to the terminal view (currently only available in GSD live status polling).
- Keyboard shortcuts: consider `useHotkeys` pattern or raw `keydown` listener on `document` with event.preventDefault for captured shortcuts.
- No new npm dependencies beyond `xterm-addon-search` (already in xterm.js ecosystem).

**Acceptance criteria for milestone:**
1. Permission prompts are visually flagged within 5 seconds of appearing in any active session
2. Context pressure is visible in the terminal view (not just GSD tab) when above threshold
3. Ctrl+F opens terminal search and finds text in the scrollback buffer
4. Keyboard shortcuts exist for tab switching and terminal focus
5. All features work in production build (not just dev mode)
6. Playwright E2E tests cover search overlay and keyboard shortcuts

**Out of scope for this milestone:**
- Mobile terminal improvements (deferred — xterm.js mobile touch still broken)
- Multi-pane terminal layouts
- Agent creation/management
- Real-time collaboration features
