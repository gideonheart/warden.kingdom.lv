### Phase 37: Crash Detection Backend
- **Metadata gap:** 37-01-SUMMARY.md missing `requirements-completed: [CRSH-01, CRSH-02]` in YAML frontmatter. Uses older template format. Both requirements fully satisfied per VERIFICATION.md evidence.

### Phase 38: Auto-Restart Engine
- **UX note:** `once` restart policy mode is functionally identical to `always` — both restart on every crash up to the 3/hr storm limit. This is by explicit plan design (`38-02-PLAN.md:118`: "once means restart once per crash event, NOT once ever"). CRSH-03 requirement does not specify self-revert behavior. **Recommendation:** Add tooltip in `RestartPolicyDropdown` explaining semantics.
