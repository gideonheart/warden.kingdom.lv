**1. Add shared types to `src/shared/types.ts`:**

Append the following interfaces after the existing `TokenUsageRow` interface:

```typescript
export type BurnWindow = 'today' | '2day' | '7day';

export interface BurnRateEntry {
  agentId: string;
  windowCostUsd: number;
  burnRatePerHour: number;
  projectedDailyUsd: number;
  projectedWeeklyUsd: number;
}

export interface BudgetConfig {
  agentId: string;
  dailyBudgetUsd: number;
}

export interface BudgetAlertStatus {
  agentId: string;
  todayCostUsd: number;
  dailyBudgetUsd: number;
  budgetPct: number;
  alertLevel: 'ok' | 'warning' | 'exceeded';
}
```

Per CONTEXT.md locked decision: window labels are "Today", "2-day", "7-day" — NOT "1h/4h/24h". The `BurnWindow` type uses string literals matching the data reality (day-granular windows).

**2. Add `budget_config` table migration to `DatabaseConnection.ts`:**

In `runMigrations()`, after the existing `ALTER TABLE` try/catch block for cache columns, add:

```typescript
this.db.exec(`
  CREATE TABLE IF NOT EXISTS budget_config (
    agent_id TEXT PRIMARY KEY,
    daily_budget_usd REAL NOT NULL DEFAULT 0,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
  )
`);
```

Follow the existing inline migration pattern — no separate migration runner.

**3. Add database methods to `DatabaseConnection` class:**

Import `BurnRateEntry`, `BudgetConfig`, `BudgetAlertStatus`, and `BurnWindow` from `../../shared/types.js`.

Add the following methods:

a) `getBurnRate(window: BurnWindow): BurnRateEntry[]` — Maps window to day lookback: `today` -> 1 day/24h, `2day` -> 2 days/48h, `7day` -> 7 days/168h. Queries `token_usage` with `WHERE date >= date('now', '-N days')`, groups by `agent_id`, computes `SUM(cost_usd)/hours` as `burnRatePerHour`, `burnRatePerHour*24` as `projectedDailyUsd`, `burnRatePerHour*168` as `projectedWeeklyUsd`. Returns array ordered by burnRatePerHour DESC. Interpolate numeric constants into SQL (not user input — safe from injection).

b) `getAllBudgetConfigs(): BudgetConfig[]` — Simple `SELECT agent_id as agentId, daily_budget_usd as dailyBudgetUsd FROM budget_config WHERE daily_budget_usd > 0 ORDER BY agent_id`.

c) `upsertBudgetConfig(agentId: string, dailyBudgetUsd: number): void` — INSERT with ON CONFLICT DO UPDATE (upsert). If `dailyBudgetUsd` is 0, DELETE the row instead (clearing a budget removes it entirely per CONTEXT.md: "Remove budget by clearing to $0 or empty").

d) `getBudgetAlertStatus(): BudgetAlertStatus[]` — LEFT JOIN `budget_config` against `token_usage` for today's date (`tu.date = date('now')`). Only returns agents with `daily_budget_usd > 0`. Computes `budgetPct = COALESCE(tu.cost_usd, 0) / bc.daily_budget_usd * 100`. Derives `alertLevel`: `>= 100` -> 'exceeded', `>= 80` -> 'warning', else 'ok'. Ordered by budgetPct DESC.

Add four new endpoints to `historyRoutes.ts`. Import `BurnWindow` from `../../shared/types.js`.

**1. GET /api/history/burn-rate**

Query parameter: `window` (string, optional, defaults to `'today'`).

Validate: must be one of `'today'`, `'2day'`, `'7day'`. If invalid, default to `'today'` (per CONTEXT.md: "Default window on view open: Today").

Call `database.getBurnRate(window)`. Return JSON: `{ burnRates: BurnRateEntry[], window: string }`.

**2. GET /api/history/budget-config**

No parameters. Call `database.getAllBudgetConfigs()`. Return JSON: `{ configs: BudgetConfig[] }`.

**3. PUT /api/history/budget-config/:agentId**

Body: `{ dailyBudgetUsd: number }`.

Validate server-side: `dailyBudgetUsd` must be a finite number >= 0. If invalid, return 400 with `{ error: 'dailyBudgetUsd must be a non-negative number' }`.

Call `database.upsertBudgetConfig(agentId, value)`. Return JSON: `{ status: 'ok' }`.

**4. GET /api/history/budget-config/status**

No parameters. Call `database.getBudgetAlertStatus()`. Compute aggregate `alertLevel` from the array: if any status is `'exceeded'` -> `'exceeded'`, else if any is `'warning'` -> `'warning'`, else `'ok'`. Return JSON: `{ alertLevel: string, statuses: BudgetAlertStatus[] }`.

**Important:** The `/budget-config/status` route MUST be registered BEFORE `/budget-config/:agentId` to prevent Express from treating `status` as an `:agentId` parameter. Place the status GET route first, then the PUT route with `:agentId`.
