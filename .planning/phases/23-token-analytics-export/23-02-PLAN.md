  <action>
Create a new component `src/client/components/ModelComparisonView.tsx`:

1. **Props interface:** `{ agentFilter?: string }` — optional agent filter from parent.

2. **Time range state:** State variable `timeRange` with values `'24h' | '7d' | '30d' | 'all'`, default `'7d'` per CONTEXT.md. Preset buttons row at the top with styling matching the existing BURN_WINDOWS selector pattern in TokenUsageView (segmented button group with warden theme tokens).

3. **Data fetching:** `useEffect` fetches `GET /api/history/model-comparison` with query params:
   - `agentId` from props (if set)
   - `dateFrom` calculated from timeRange: 24h = today's date, 7d = 7 days ago, 30d = 30 days ago, all = omit. Format as YYYY-MM-DD.
   - `dateTo` omitted (defaults to all data up to now)
   Re-fetches when timeRange or agentFilter changes.

4. **Model color mapping:** Define a `MODEL_COLORS` map that assigns colors by model name substring:
   - If model name contains `opus` → purple (`bg-purple-500`)
   - If model name contains `sonnet` → blue (`bg-blue-500`)
   - If model name contains `haiku` → green (`bg-emerald-500`)
   - Fallback → `bg-warden-accent/70` (gray/neutral)

5. **Insight headline:** Calculate the most expensive model across all agents: sum totalCostUsd by model, find the max, compute its percentage of grand total. Display as: "{Model} is {pct}% of total spend" or "No usage data yet" if empty. The headline should be bold text at the top of the chart area, providing at-a-glance actionable info per CONTEXT.md.

6. **Bar chart rendering:** Group data by agent. For each agent:
   - Section header with `formatAgentId(agentId)` (reuse the same function pattern from TokenUsageView — define locally or import)
   - Per-agent total cost label on the right of the header
   - Horizontal bars for each model within the agent, using global max across all data for bar width scaling (global scale per research recommendation)
   - Each bar: model name label (truncated, 140px), colored bar div using MODEL_COLORS, cost label right-aligned (`$X.XXXX` with 4 decimal places)
   - Bar height: `h-5` for comfortable spacing

7. **Empty state:** When `data.length === 0 && !isLoading`, show "No model comparison data yet — data appears after a token usage scan runs."

8. **Loading state:** Simple "Loading..." text or a brief skeleton while fetching.

Use `warden-*` theme tokens throughout: `bg-warden-panel`, `text-warden-text`, `text-warden-text-dim`, `border-warden-border` for structural elements. Model bar colors are the exception — they use distinct palette colors per CONTEXT.md.

Include a small download icon (use a simple SVG inline or Unicode character like "⬇" if no icon library).

5. **Export handler:** `handleExport` callback:
   ```typescript
   const handleExport = useCallback(async () => {
     try {
       const response = await fetch('/api/history/token-usage/export');
       if (!response.ok) throw new Error('Export failed');
       const blob = await response.blob();
       const url = URL.createObjectURL(blob);
       const anchor = document.createElement('a');
       anchor.href = url;
       const today = new Date().toISOString().slice(0, 10);
       anchor.download = `warden-token-usage-${today}.csv`;
       anchor.click();
       URL.revokeObjectURL(url);
       setExportToast(true);
       setTimeout(() => setExportToast(false), 3000);
     } catch (error) {
       console.error('[TokenUsageView] Export failed:', error);
     }
   }, []);
   ```

6. **Toast notification:** Add state `exportToast: boolean`, default false. When true, render a small toast at the bottom-right or top-right of the component: "Export downloaded" with `bg-warden-accent/20 text-warden-accent text-sm px-3 py-2 rounded` styling. Auto-dismiss after 3 seconds via setTimeout in handleExport. Use Tailwind transition classes for smooth appearance.

IMPORTANT: Preserve ALL existing TokenUsageView functionality — burn rates, budget editor, window selector, projection card. The tab just wraps the existing content. Do NOT move or remove anything.
