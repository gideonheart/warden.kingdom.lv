**CSV export endpoint** — register BEFORE the POST `/api/history/token-usage/scan` to avoid any Express path ambiguity, though `/export` vs `/scan` are unambiguous:

```typescript
historyRoutes.get('/api/history/token-usage/export', (request, response) => {
  const rows = database.getTokenUsageForExport({});

  const headers = ['date', 'agent_id', 'model', 'input_tokens', 'output_tokens',
                   'cache_creation_input_tokens', 'cache_read_input_tokens', 'cost_usd'];
  const csvLines = [
    headers.join(','),
    ...rows.map((row) =>
      [row.date, row.agentId, row.model, row.inputTokens, row.outputTokens,
       row.cacheCreationInputTokens, row.cacheReadInputTokens, row.costUsd.toFixed(6)].join(',')
    ),
  ];
  const csvContent = csvLines.join('\n');

  const today = new Date().toISOString().slice(0, 10);
  response.setHeader('Content-Type', 'text/csv; charset=utf-8');
  response.setHeader('Content-Disposition', `attachment; filename="warden-token-usage-${today}.csv"`);
  response.send(csvContent);
});
```

The CSV always exports the full unfiltered dataset per CONTEXT.md decision. The filename includes the export date per CONTEXT.md: `warden-token-usage-YYYY-MM-DD.csv`.

Note: No need to import anything new — `database` is already imported. The export row type flows through the database method.
