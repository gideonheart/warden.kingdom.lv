  - For sessions in `'stopping'` state: if the tmux session no longer exists, update status to `'stopped'`. If still exists after 15s, force-kill via `tmuxSessionManager.destroySession()` then mark `'stopped'`.
  - Keep existing logic for active/idle sessions unchanged.

**src/server/routes/instanceRoutes.ts:**
- Import `openClawConfigReader` from `../services/OpenClawConfigReader.js` for agent config lookup.
- Import `spawn` from `child_process` and `openSync`/`closeSync` from `fs` for fire-and-forget session creation (following the GSD spawn pattern from gsdRoutes.ts).

**POST /api/instances/start** endpoint:
  1. Accept JSON body `{ agentId: string }`. Validate agentId is non-empty string.
  2. Check for duplicate: call `instanceTracker.findActiveByAgentId(agentId)`. If found, return `409 { error: 'Agent already has an active session', existingInstance }`.
  3. Look up agent config: `openClawConfigReader.getAgents()` to find workspace path. If agent not found in config, return `404 { error: 'Agent not found in openclaw.json' }`.
  4. Derive `projectPath` from agent's workspace (resolve relative paths against `$HOME/.openclaw/`). Derive `projectSlug` from the project path (last directory segment).
  5. Build session name via `tmuxSessionManager.buildSessionName(agentId, projectSlug)`.
  6. Pre-register instance in DB with status `'starting'`: call `database.upsertInstance({ agentId, agentName, tmuxSessionName, projectPath, telegramTopicId: undefined })` then immediately `database.updateInstanceStatus(newInstance.id, 'starting')`.
  7. Fire-and-forget the actual tmux+claude creation: use `spawn` with a simple inline shell script that runs tmux new-session then sends claude command. Detach with `{ detached: true, stdio: ['ignore', logFd, logFd] }` and `child.unref()`. Log to `/tmp/warden-start-${agentId}.log`.
     Alternative (simpler): Just call `tmuxSessionManager.createSessionWithClaude(agentId, projectSlug, projectPath)` in a `.then()/.catch()` promise chain — if it succeeds, update status to `'active'`; if it fails, update to `'error'`. This avoids spawn complexity since tmux commands are fast (< 1s).
  8. Return `202 { message: 'Starting agent session', instance }` immediately.

**Update POST /api/instances/:id/stop** endpoint (replace existing):
  1. Find instance by ID. If not found, 404.
  2. If instance status is already `'stopped'` or `'stopping'`, return `409 { error: 'Session is already stopped or stopping' }`.
  3. Set instance status to `'stopping'` immediately (optimistic UI).
  4. Begin graceful shutdown sequence:
     a. Send Ctrl+C to the session: `tmuxSessionManager.sendCtrlC(instance.tmuxSessionName)`.
     b. Wait up to 5 seconds, polling every 500ms to check if tmux session has exited on its own: `tmuxSessionManager.sessionExists(sessionName)`.
     c. If session exits during grace period, update status to `'stopped'` and return success.
     d. If session still exists after 5s, force kill: `tmuxSessionManager.destroySession(sessionName)`, then update status to `'stopped'`.
  5. Return `200 { success: true, instance, forcedKill: boolean }`.

**POST /api/instances/:id/restart** endpoint:
  1. Find instance by ID. If not found, 404.
  2. Extract `agentId` and `projectPath` from the existing instance record.
  3. If instance is active/idle/starting, perform stop first (call the stop logic inline — send Ctrl+C, grace period, kill).
  4. Then trigger a start with the same agentId (reuse the start logic).
  5. Return `202 { message: 'Restarting agent session', instance }`.

**POST /api/instances/:id/force-kill** endpoint:
  1. Find instance by ID. If not found, 404.
  2. Skip grace period — immediately `tmuxSessionManager.destroySession(sessionName)`.
  3. Update status to `'stopped'`.
  4. Return `200 { success: true, instance }`.
  Per user decision: "Force Kill button appears during the grace period so operator can skip waiting."
