Create `src/server/services/GsdRegistryService.ts` as a singleton class mirroring the OpenClawConfigReader TTL cache pattern.

**Types to define inline (not in shared/types.ts — these are server-internal):**
- `RegistryAgent`: `{ agent_id: string; enabled: boolean; working_directory: string; tmux_session_name: string; claude_launch_command: string; auto_wake: boolean; topic_id: number; openclaw_session_id: string; claude_resume_target: string; claude_post_launch_mode: string; }`
- `GsdRegistry`: `{ global_status_openclaw_session_id: string; global_status_openclaw_session_key: string; agents: RegistryAgent[]; }`

**Class: GsdRegistryService**
- Private state: `cachedRegistry: GsdRegistry | null = null`, `lastReadAt: number = 0`
- Constant: `CACHE_TTL_MS = 30_000`
- `async getRegistry(): Promise<GsdRegistry>` — check TTL, read + parse if stale, return cached
- `async getAgent(agentId: string): Promise<RegistryAgent | undefined>` — convenience method
- `async patchAgent(agentId: string, patch: Partial<Pick<RegistryAgent, 'enabled'>>): Promise<RegistryAgent>` — read current registry, find agent by agent_id, merge patch, write atomically (writeFile to `.tmp` then fs.rename), invalidate cache (set cachedRegistry = null, lastReadAt = 0), return updated agent entry. Throw if agent not found.

**Atomic write pattern:** `writeFile(REGISTRY_PATH + '.tmp', ...)` then `rename(REGISTRY_PATH + '.tmp', REGISTRY_PATH)`. This is POSIX-atomic on same filesystem.

**Imports:** `readFile`, `writeFile`, `rename` from `fs/promises`; `path` from `path`.

**Export:** `export const gsdRegistryService = new GsdRegistryService();`

<task type="auto">
  <name>Task 2: Create GsdHookLogWatcher with fs.watchFile and Socket.IO namespace</name>
  <files>src/server/services/GsdHookLogWatcher.ts</files>
  <action>
Create `src/server/services/GsdHookLogWatcher.ts` as a singleton class mirroring the TerminalStreamService namespace pattern.

**Log path:** `/tmp/gsd-hooks.log`
**Backfill count:** 20 events (but read last 200 lines to be safe — events average 5.7 lines each)

**Class: GsdHookLogWatcher**
- Private state: `currentOffset: number = 0`, `socketServer: SocketIOServer | null = null`, `isWatching: boolean = false`
- `setupSocketNamespace(socketServer: SocketIOServer): void` — store socketServer reference, create namespace via `socketServer.of('/gsd-hooks')`, register `connection` handler that:
  - Logs connection
  - Calls `readLastLines(200)` and emits `gsd-hooks:backfill` with `{ lines }` to the connecting socket
  - Registers `disconnect` handler that logs disconnection
- `startWatching(): void` — use `fs.watchFile(HOOK_LOG_PATH, { interval: 1000, persistent: false })` (NOT `fs.watch` — watchFile works when file does not yet exist, per research Pitfall 2). On change, call `readNewLines()`. Set initial offset from `fs.statSync` in try-catch (default 0 if file missing). Set `isWatching = true`.
- `stopWatching(): void` — call `fs.unwatchFile(HOOK_LOG_PATH)`, set `isWatching = false`
- Private `readNewLines(): void` — statSync to get current size, skip if size <= currentOffset (handles truncation and dedup per research Pitfall 1). Open file descriptor, read from currentOffset to current size, close fd. Split by newline, filter empty lines. If new lines exist and socketServer is set, emit `gsd-hooks:lines` with `{ lines }` to namespace.
- Private `readLastLines(lineCount: number): string[]` — try-catch around statSync. Read last `Math.min(fileSize, lineCount * 120)` bytes from end of file. Split by newline, filter empty, return last `lineCount` entries. Return empty array on any error.

**Use synchronous fs for readNewLines and readLastLines** (fs.openSync, fs.readSync, fs.closeSync, fs.statSync) — these are small reads on a local file, and the watchFile callback expects synchronous completion.

**Imports:** `* as fs` from `fs`; `Server as SocketIOServer` from `socket.io`.

**Export:** `export const gsdHookLogWatcher = new GsdHookLogWatcher();`
  </action>
  <verify>
Run `npx tsc --noEmit` and confirm no type errors. Visually inspect that:
1. Uses `fs.watchFile` (not `fs.watch`) with interval 1000
2. startWatching wraps statSync in try-catch for missing file
3. readNewLines checks `stat.size <= this.currentOffset` before reading
4. setupSocketNamespace creates `/gsd-hooks` namespace
5. Backfill emits to individual socket, live events emit to namespace
  </verify>
  <done>GsdHookLogWatcher singleton exists, watches /tmp/gsd-hooks.log via fs.watchFile, emits new lines to /gsd-hooks Socket.IO namespace, backfills last 200 lines on connect, handles missing log file gracefully, and passes TypeScript type checking.</done>
