# Phase 11.1 Plan 01: Fix Terminal Visibility When Mobile Keyboard Opens Summary

**visualViewport resize listener in TerminalView refits xterm.js terminal on iOS keyboard open/close; MobilePromptSheet dynamically offsets bottom to float above keyboard**

## Accomplishments
- Terminal (xterm.js FitAddon) now refits when iOS Safari fires visualViewport.resize on keyboard open/close, keeping content visible
- MobilePromptSheet floats above the iOS keyboard by calculating layout vs visual viewport offset and setting style.bottom dynamically
- Both changes clean up listeners on unmount; no memory leaks or stale handlers
- Production build passes, TypeScript clean, no regressions to desktop window.resize behavior

## Files Created/Modified
- `src/client/components/TerminalView.tsx` - Replaced handleWindowResize with debounced refitTerminal listening to window.resize + visualViewport.resize; cleans up both on unmount
- `src/client/components/MobilePromptSheet.tsx` - New useEffect tracks visualViewport resize+scroll, computes offsetFromLayoutBottom, sets sheet.style.bottom dynamically

## Decisions Made
- 100ms debounce on refitTerminal to prevent FitAddon collapse bug (xterm.js #5320)
- requestAnimationFrame wrapping ensures DOM measurements happen after layout settles
- Listen to BOTH resize and scroll on visualViewport in MobilePromptSheet — iOS can fire scroll events when keyboard animates in
- MobileKeyToolbar uses `sticky` (already correct), no changes needed there
