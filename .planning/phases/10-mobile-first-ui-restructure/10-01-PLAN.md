    **1. Mobile header in App.tsx:**
    Add a mobile-responsive header. On screens below `sm:` (640px), show only the active view name as a label and a hamburger button that toggles a dropdown menu. The dropdown contains: Terminals, History, Plugins, Agents toggle, Refresh. On `sm:` and above, keep the current inline button layout.

    Implementation: Add a `[showMobileMenu, setShowMobileMenu]` boolean state. In the header, wrap the existing nav buttons in a `hidden sm:flex` div. Add a new mobile-only button (`sm:hidden`) that shows a three-line hamburger icon (use `\u2630` character or three horizontal lines via CSS). When tapped, show an absolute-positioned dropdown below the header with all nav options. Each option is a button with `min-h-[44px]` touch target. Tapping an option calls the appropriate handler and closes the menu. Add a click-outside handler to dismiss.

    **2. Safe area insets in styles.css:**
    Add `padding-bottom: env(safe-area-inset-bottom, 0px)` to:
    - The `.app-height` class (for main layout)
    - A new `.safe-bottom` utility class

    Add `padding-left: env(safe-area-inset-left, 0px)` and `padding-right: env(safe-area-inset-right, 0px)` to the header in App.tsx.

    Add `<meta name="viewport" content="width=device-width, initial-scale=1, viewport-fit=cover">` — check `index.html` and update if the `viewport-fit=cover` part is missing (required for safe area insets to work).

    **3. Touch targets audit:**
    Ensure ALL interactive elements have `min-h-[44px]` minimum:
    - App.tsx: All nav buttons (Terminals, History, Plugins) — add `min-h-[44px]` to each. The Agents button already has it.
    - AgentSidebar.tsx: Agent selection buttons — add `min-h-[44px]` to the button className
    - HistoryView.tsx: Tab buttons (Sessions, Token Usage, Gateway Logs) — add `min-h-[44px]` to each

    **4. Scroll-snap tabs in InstanceTabBar.tsx:**
    Add CSS scroll-snap to the tab container div:
    - Container: add classes `snap-x snap-mandatory` (Tailwind CSS v4 scroll-snap utilities)
    - Each tab item div: add class `snap-start`

    If Tailwind v4 does not support `snap-x` as a utility, add the following to styles.css instead:
    ```css
    .tab-snap { scroll-snap-type: x mandatory; }
    .tab-snap > * { scroll-snap-align: start; }
    ```
    And apply `tab-snap` class to the tab container.

    **5. Safe area on mobile key toolbar (TerminalView.tsx MobileKeyToolbar):**
    Add `safe-bottom` class (or `pb-[env(safe-area-inset-bottom)]` if Tailwind supports it) to the MobileKeyToolbar outer div so the toolbar sits above the home indicator on iPhone.
