**Problem:** `selectedAgentId` in App.tsx is only set by `handleSelectAgent` (AgentSidebar click). When user clicks a session tab, `handleSelectSession` updates `selectedSessionName` but never derives the agent ID. The `PromptPanel` receives a stale/null `selectedAgentId`, so the dropdown doesn't sync and `effectiveAgentId` is empty, disabling Send.

**Fix in App.tsx:**

1. Remove the standalone `selectedAgentId` state variable and `handleSelectAgent` callback â€” they are only used for the sidebar and should not be the source of truth for the prompt panel.

2. Add a new derived value that computes the agent ID from the currently selected session. After the `activeInstances` filter (line 21-23), add:

```typescript
// Derive agent ID from selected session for prompt panel sync
const selectedInstance = activeInstances.find(
  (instance) => instance.tmuxSessionName === selectedSessionName
);
const derivedAgentId = selectedInstance?.agentId ?? null;
```

3. Pass `derivedAgentId` to `PromptPanel` instead of `selectedAgentId`:

Change line 147:
```typescript
<PromptPanel agents={agents} selectedAgentId={derivedAgentId} />
```

4. For `AgentSidebar`, keep its own local state if needed. The sidebar's `selectedAgentId` is cosmetic (highlight in sidebar) and should remain independent. Keep the existing `selectedAgentId` state + `handleSelectAgent` for sidebar only, but rename to `sidebarSelectedAgentId` / `handleSidebarSelectAgent` for clarity:

```typescript
const [sidebarSelectedAgentId, setSidebarSelectedAgentId] = useState<string | null>(null);
const handleSidebarSelectAgent = useCallback((agentId: string) => {
  setSidebarSelectedAgentId(agentId);
}, []);
```

And update AgentSidebar props to use `sidebarSelectedAgentId` and `handleSidebarSelectAgent`.

**Fix in PromptPanel.tsx:**

1. Add `useEffect` import (already imported from React).

2. Add a `useEffect` to sync `targetAgentId` when `selectedAgentId` prop changes (tab switch). This resets the manual override on every tab switch per the recommended behavior from research:

```typescript
import { useState, useCallback, useEffect } from 'react';
```

After the existing state declarations, add:

```typescript
// Sync dropdown with tab selection - resets manual override on tab switch
useEffect(() => {
  if (selectedAgentId) {
    setTargetAgentId(selectedAgentId);
  }
}, [selectedAgentId]);
```

This means: every time the user switches tabs (which changes `selectedAgentId` via `derivedAgentId`), the dropdown resets to match. Manual override (changing dropdown directly) persists until the next tab switch.

3. Ensure `effectiveAgentId` fallback chain is correct (it already is at line 15):
```typescript
const effectiveAgentId = targetAgentId || selectedAgentId || agents[0]?.id || '';
```

This handles the case where no session exists but agents are configured â€” falls back to first agent.

**Key points:**
- Do NOT use `useEffectEvent` â€” not available in React 19.0.0
- Do NOT include `targetAgentId` in the useEffect dependency array (would cause infinite loop)
- The `handleSend` useCallback already depends on `effectiveAgentId` (line 44), so it will pick up changes correctly
- The `handleKeyDown` depends on `handleSend` (line 53), so Ctrl+Enter also gets the updated agent
