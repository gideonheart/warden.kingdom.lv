**3. Create `src/server/services/ActivityEventService.ts`:**

This is a singleton service class with these responsibilities:

a) **ANSI stripping** -- Inline the `ansi-regex@5` pattern as a module-level constant. Do NOT import `strip-ansi` (CJS/ESM incompatibility). The regex pattern (from research):
```typescript
const ANSI_PATTERN = [
  '[\\u001B\\u009B][[\\]()#;?]*(?:(?:(?:(?:;[-a-zA-Z\\d\\/#&.:=?%@~_]+)*|[a-zA-Z\\d]+(?:;[-a-zA-Z\\d\\/#&.:=?%@~_]*)*)?\\u0007)',
  '(?:(?:\\d{1,4}(?:;\\d{0,4})*)?[\\dA-PR-TZcf-ntqry=><~]))',
].join('|');
const ANSI_REGEX = new RegExp(ANSI_PATTERN, 'g');
function stripAnsi(input: string): string {
  return typeof input === 'string' ? input.replace(ANSI_REGEX, '') : input;
}
```

b) **Chunk buffering** -- `Map<string, string>` keyed by sessionName. Buffer incoming ANSI-stripped text. Only parse complete lines (terminated by `\n`). Flush buffer if it exceeds 4KB. Clear buffer on session disconnect via `clearSessionBuffer(sessionName)`.

c) **Pattern matching** for Claude Code output (from research, verified from live capture):
- Tool call: `/^[●⏺]\s+([\w]+)\((.{0,200})\)/m` -> event_type `tool_call`, summary = `{ToolName}({truncated args})`
- File edit result: `/^[⎿]\s+(?:Updated|Created|Wrote)\s+(.{1,200})/m` -> event_type `file_edit`, success=true
- Error result: `/^[⎿]\s+Error:/m` -> event_type `error`, success=false
- Bash exit code: `/exit code[:\s]+(\d+)/i` -> event_type `bash_command`, success = (code === 0)

d) **Operator input batching** -- `Map<string, { buffer: string; timer: ReturnType<typeof setTimeout> }>` keyed by sessionName. Accumulate keystrokes. Flush on `\n` (Enter) or after 2 seconds of inactivity. Skip pure control sequences (length === 1 and charCode < 32, except \n and \r). Store truncated to 500 chars.

e) **Instance ID resolution** -- When inserting events for terminal-sourced events, look up `database.findInstanceBySessionName(sessionName)?.id`. Cache the result per session in a Map to avoid repeated DB lookups.

f) **Public methods:**
- `captureSessionStart(sessionName: string, agentId: string, instanceId: number | null): void`
- `captureSessionStop(sessionName: string, agentId: string): void`
- `capturePromptSent(agentId: string, sessionName: string, prompt: string, success: boolean): void`
- `captureOperatorInput(sessionName: string, agentId: string, input: string): void` -- feeds into the input batcher
- `processTerminalChunk(sessionName: string, agentId: string, rawChunk: string): void` -- strips ANSI, buffers, parses complete lines
- `clearSessionBuffer(sessionName: string): void` -- clears chunk buffer and flushes input buffer
- `startRetentionCleanup(): void` -- sets up `setInterval` to call `purgeOldActivityEvents` every 6 hours
- `stopRetentionCleanup(): void` -- clears the interval

Export as singleton: `export const activityEventService = new ActivityEventService();`

**Important:** All database writes in `processTerminalChunk` must be wrapped in try/catch to prevent terminal output pipeline crashes. Log errors with `console.error('[ActivityEvent]', ...)` but never throw.
