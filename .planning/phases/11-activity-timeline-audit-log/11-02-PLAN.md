**2. Create `src/client/components/ActivityView.tsx`:**

Follow the exact same pattern as `SessionHistory.tsx` for filters, pagination, loading states.

Props:
```typescript
interface ActivityViewProps {
  onNavigateToSession?: (sessionName: string) => void;
}
```

State:
```typescript
interface ActivityFilters {
  agentId: string;
  eventType: string;
  dateFrom: string;
  dateTo: string;
}
```

- `events: ActivityEvent[]`, `total: number`, `page: number`, `isLoading: boolean`, `filters: ActivityFilters`
- `eventTypes: string[]` -- fetched once on mount from `GET /api/activity/event-types`
- `pageSize = 50`

Implementation:

a) **Data fetching**: `useCallback` + `useEffect` pattern identical to SessionHistory. Fetch from `GET /api/activity/events?agentId=...&eventType=...&dateFrom=...&dateTo=...&limit=50&offset=N`. Parse response as `{ events, total }`.

b) **Event types fetch**: On mount, fetch `GET /api/activity/event-types` and populate `eventTypes` state for the dropdown.

c) **Filter bar**: `flex flex-wrap gap-3` container with:
  - Agent ID text input: same styling as SessionHistory (`bg-warden-bg border border-warden-border rounded px-2 py-1 min-h-[44px] text-sm text-warden-text w-32`). Placeholder "Agent ID".
  - Event type select dropdown: populated from `eventTypes` state. First option `<option value="">All types</option>`, then map each type to a human-readable label. Same styling as SessionHistory select.
  - Date from/to inputs: `type="date"`, same styling as SessionHistory.
  - Total count: `text-xs text-warden-text-dim self-center` showing `{total} events`.
  - **Export buttons**: Two small buttons after the count:
    - "CSV" button: `px-2 py-1 min-h-[44px] text-xs bg-warden-border/50 text-warden-text-dim hover:text-warden-text rounded transition-colors`
    - "JSON" button: same styling
    - Both disabled when `events.length === 0` with `disabled:opacity-30`

d) **Export functions** (defined inside the component or as module-level helpers):

`exportToCSV(events: ActivityEvent[])`:
- Headers: `id,timestamp,agentId,sessionName,eventType,summary,success,detail`
- For each event, produce comma-separated row. Escape `summary` and `detail` by wrapping in double quotes and doubling any internal quotes.
- Create `Blob` with BOM prefix `\uFEFF` and type `text/csv;charset=utf-8`.
- Generate filename: `warden-activity-${new Date().toISOString().slice(0,10)}.csv`
- Trigger download via temporary anchor element: `URL.createObjectURL`, set `anchor.download`, click, cleanup.

`exportToJSON(events: ActivityEvent[])`:
- `JSON.stringify(events, null, 2)`
- Create `Blob` with type `application/json`.
- Filename: `warden-activity-${new Date().toISOString().slice(0,10)}.json`
- Same download trigger pattern.

**Important on export scope**: Export ALL currently filtered events, not just the current page. When export is clicked, make a separate fetch with `limit=10000&offset=0` using the current filters to get the full dataset. Show a brief "Exporting..." state if needed.

e) **Event list**: Loading spinner (same as SessionHistory), empty state ("No activity events found"), then map `events` to `<ActivityEventRow>` components. Pass `onNavigateToSession` prop through.

f) **Pagination**: Same Previous/Next pattern as SessionHistory. Show `Page X of Y`.

g) All interactive elements must have `min-h-[44px]` for mobile touch targets.
