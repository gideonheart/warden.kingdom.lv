  <name>Task 1: Create useGsdRegistry and useGsdHookFeed data hooks</name>
  <files>src/client/hooks/useGsdRegistry.ts, src/client/hooks/useGsdHookFeed.ts</files>
  <action>
Create two custom hooks following existing codebase patterns (useActiveInstances for polling, useTerminalSocket for Socket.IO).

**useGsdRegistry.ts** (~60-70 LOC):
- Define `RegistryAgent` interface inline (matching the server's GsdRegistry shape from 12-RESEARCH.md): `agent_id`, `enabled`, `working_directory`, `tmux_session_name`, `claude_launch_command`, `auto_wake`, `topic_id`, `openclaw_session_id`, `claude_resume_target`, `claude_post_launch_mode`.
- Define `GsdRegistry` interface with `global_status_openclaw_session_id`, `global_status_openclaw_session_key`, `agents: RegistryAgent[]`.
- `useGsdRegistry()` hook:
  - State: `registry: GsdRegistry | null`, `isLoading: boolean`, `error: string | null`, `optimisticEnabled: Record<string, boolean>` (overlay for optimistic toggles).
  - `fetchRegistry`: GET `/api/gsd/registry`, set registry + clear error on success, set error on failure, set isLoading false in finally.
  - Poll every 10_000ms (registry changes infrequently). Use `useEffect` + `setInterval` + cleanup pattern identical to `useActiveInstances.ts`.
  - `toggleEnabled(agentId: string, currentEnabled: boolean)`: Immediately set `optimisticEnabled[agentId] = !currentEnabled`. PATCH `/api/gsd/registry/agents/${agentId}` with `{ enabled: !currentEnabled }`. On success: remove agentId from optimisticEnabled (let poll data catch up). On error: revert optimisticEnabled[agentId] = currentEnabled.
  - `getEffectiveEnabled(agentId: string, serverEnabled: boolean)`: Returns `optimisticEnabled[agentId] ?? serverEnabled`. Used by UI to read the visually correct value.
  - Export: `{ registry, isLoading, error, refetch: fetchRegistry, toggleEnabled, getEffectiveEnabled }`.

**useGsdHookFeed.ts** (~80-90 LOC):
- Define `HookEvent` interface: `timestamp: string`, `hookScript: string`, `hookEventName: string`, `agentId: string | null`, `tmuxSession: string | null`, `state: string | null`.
- `parseHookEvents(lines: string[], maxEvents: number): HookEvent[]`:
  - Iterate lines. On `FIRED` regex match (`/^\[(.+?)\] \[(.+?)\] FIRED/`), push previous event (if any) and start new one.
  - Extract fields from continuation lines: `hook_event_name=(\S+)`, `agent_id=(\S+)`, `tmux_session=(\S+)`, `state=(\S+)`.
  - Return last `maxEvents` events reversed (newest first).
- `useGsdHookFeed()` hook:
  - State: `hookLines: string[]` (raw lines buffer, keep last 200).
  - `useEffect`: Create `io('/gsd-hooks', { reconnection: true, reconnectionDelay: 2_000, reconnectionAttempts: 10 })`.
  - Listen for `gsd-hooks:backfill` (`{ lines: string[] }`) — set hookLines to `lines.slice(-200)`.
  - Listen for `gsd-hooks:lines` (`{ lines: string[] }`) — append and slice to last 200.
  - Cleanup: `socket.disconnect()`.
  - `hookEvents = useMemo(() => parseHookEvents(hookLines, 20), [hookLines])`.
  - Export: `{ hookEvents, hookLines }`.
- Import `io` from `socket.io-client`.
  </action>
  <verify>
Run `npx tsc --noEmit` — no type errors in the two new hook files. Verify each file exists, exports the documented function, and has the correct import patterns.
  </verify>

  <action>
Create the GSD Manager plugin file following the example-plugin.tsx pattern. This is a single file with manifest + PanelComponent. The plugin auto-registers via `import.meta.glob` in `plugins/index.ts` — no other files need modification.

**Manifest:**
```typescript
const manifest = {
  id: 'gsd-manager',
  name: 'GSD Manager',
  version: '1.0.0',
  description: 'Control center for GSD agent sessions',
  slot: 'bottom-panel',
  capabilities: ['gsd-management', 'agent-control', 'session-monitoring'],
} as const satisfies PluginManifest;
```

**CopyButton helper component (~15 LOC):**
- Props: `{ text: string }`.
- State: `copied: boolean`.
- onClick: `navigator.clipboard.writeText(text).then(() => { setCopied(true); setTimeout(() => setCopied(false), 2000); }).catch(() => {})`.
- Render: `<button>` with warden theme classes. Shows "Copied!" for 2s, then "Copy".

**BashHint helper component (~8 LOC):**
- Props: `{ command: string }`.
- Renders a `<div>` containing `<code className="...">` with the bash command text and a `<CopyButton text={command} />`.
- Use `text-xs text-warden-text-dim font-mono bg-warden-bg/50 px-2 py-1 rounded` or similar warden-themed styling.

**GsdManagerPanel component (the PanelComponent, ~200-250 LOC):**

PanelComponent receives ZERO props by contract. All data is fetched internally.

State:
- `isExpanded: boolean` — starts `false` (collapsed). Panel starts collapsed to preserve terminal height.
- `activeTab: 'grid' | 'controls' | 'registry' | 'hooks'` — starts `'grid'`.

When collapsed: render only a thin clickable header bar (`~h-8`) with text "GSD Control Center" and an expand chevron. Use `bg-warden-panel border-t border-warden-border`. Do NOT mount any hooks when collapsed — use conditional rendering so hooks only activate when expanded.

When expanded: render full panel with tabs and content. Target height: `h-64` or `max-h-64 overflow-y-auto`.

**Hooks (called conditionally when expanded):**
- `useGsdRegistry()` — for grid, registry tab, and spawn form.
- `useGsdHookFeed()` — for hooks tab.
- `useActiveInstances()` — for session dropdown in command dispatch (imported from existing `../hooks/useActiveInstances.js`).

**Tab bar:** 4 tabs: "Agents" (grid), "Controls" (spawn + command), "Registry", "Hooks". Use `bg-warden-panel` for inactive, `bg-warden-bg` for active tab. Compact tab style — `text-xs px-3 py-1.5`.

**Tab 1: "Agents" (Agent Grid) — GRID-01, GRID-02:**
- Show a table/grid of registry agents joined with active instances.
- Columns: Status dot (use STATUS_COLORS pattern from InstanceTabBar.tsx: active=green, idle=yellow, stopped=red), Agent ID, Tmux Session, Working Dir (truncated), Enabled toggle.
- Join logic: For each `registry.agents[i]`, find matching instance from `useActiveInstances()` where `instance.tmuxSessionName === agent.tmux_session_name`. If found, use `instance.status`. If not found, show "stopped".
- Working directory: truncate with `...` prefix if longer than ~30 chars (show last 30 chars).
- Enabled toggle: call `toggleEnabled(agent.agent_id, effectiveEnabled)` from `useGsdRegistry()`.
- If registry is loading, show "Loading registry...". If error, show error text.

**Tab 2: "Controls" (Spawn + Command) — CTRL-01, CTRL-02, DX-01:**
Two forms stacked vertically with a visual separator.

Spawn form:
- Three inputs: Agent Name (`<input type="text">`), Working Directory (`<input type="text">`), First Command (`<input type="text">`, optional placeholder "Optional first command").
- State: `agentName`, `workdir`, `firstCommand`, `isSpawning`, `spawnStatus: { type: 'success'|'error', text: string } | null`.
- Submit: POST `/api/gsd/spawn` with `{ agentName, workdir, ...(firstCommand ? { firstCommand } : {}) }`. Disable button and show "Spawning..." during fetch. On 202: show success with `data.expectedSessionName`. On error: show error. Clear status after 5s.
- Below the form: `<BashHint command={`spawn.sh ${agentName} ${workdir}${firstCommand ? ` "${firstCommand}"` : ''}`} />`.

Command dispatch form:
- Two inputs: Session (`<select>` populated from `useActiveInstances().instances.filter(i => i.status === 'active' || i.status === 'idle')`), Command (`<input type="text">`).
- State: `targetSession`, `commandText`, `isDispatching`, `dispatchStatus`.
- Submit: POST `/api/gsd/sessions/${targetSession}/command` with `{ action: 'clear_then', args: commandText }`. Show "Dispatched" on success. Show error on failure.
- Below the form: `<BashHint command={`menu-driver.sh ${targetSession} clear_then "${commandText}"`} />`.

**Tab 3: "Registry" — REG-01, REG-02:**
- Table of all registry agents showing: `agent_id`, `enabled` (toggle switch), `working_directory`, `tmux_session_name`, `auto_wake`, `claude_post_launch_mode`.
