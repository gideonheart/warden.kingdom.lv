  <name>Task 1: Add live-status server endpoint with tmux pane capture</name>
  <files>src/server/routes/gsdRoutes.ts</files>
  <action>
Add the following to `gsdRoutes.ts` (above the existing routes, after the regex constants):

1. **Type definitions** (module-level):
   ```typescript
   type AgentStateHint = 'working' | 'idle' | 'menu' | 'permission_prompt' | 'error';
   type PressureLevel = 'ok' | 'warning' | 'critical';
   ```

2. **`detectAgentState(pane: string): AgentStateHint` function** (module-level):
   - Check patterns in this order (first match wins):
     - `/enter to select|numbered.*option/i` → `'menu'`
     - `/permission|allow|dangerous/i` → `'permission_prompt'`
     - `/what can i help|waiting for/i` → `'idle'`
     - Any line matching `/error|failed|exception/i` but NOT `/error handling/i` → `'error'`
     - Default → `'working'`
   - These patterns mirror the stop-hook.sh section 8 logic exactly.

3. **`extractContextPressure(pane: string)` function** (module-level):
   - Filter empty lines from pane, take last 5 non-empty lines, join with newline.
   - Match `/(\d{1,3})%/` on the joined string.
   - If no match → return `{ contextPressure: null, contextPressureLevel: null }`.
   - If match → parse integer. Level: `>=80` → `'critical'`, `>=50` → `'warning'`, else `'ok'`.
   - Return `{ contextPressure: number, contextPressureLevel: PressureLevel }`.

4. **`GET /api/gsd/agents/live-status` route**:
   - Read registry via `gsdRegistryService.getRegistry()`.
   - For each agent, if `tmux_session_name` is falsy → return nulls for state/pressure.
   - Otherwise, call `execFileAsync('tmux', ['capture-pane', '-pt', \`${agent.tmux_session_name}:0.0\`, '-S', '-5'])`.
   - Wrap each agent call in try/catch (dead sessions throw — return nulls on catch).
   - Use `Promise.allSettled` to parallelize all agent captures.
   - Response shape: `{ agents: Array<{ agentId, sessionName, state, contextPressure, contextPressureLevel }> }`.
   - Wrap entire handler in try/catch → 500 on unexpected errors.

Note: `execFileAsync` is already imported at the top of gsdRoutes.ts via `const execFileAsync = promisify(execFile)`.
  </action>
  <verify>

```bash
curl -s http://localhost:3001/api/gsd/agents/live-status | jq .
```
Should return JSON with `agents` array. Active sessions should have non-null state and contextPressure. Stopped sessions should have null values.
  </verify>
  <done>
`GET /api/gsd/agents/live-status` returns a JSON array with `agentId`, `sessionName`, `state` (one of: working/idle/menu/permission_prompt/error or null), `contextPressure` (number 0-100 or null), and `contextPressureLevel` (ok/warning/critical or null) for each registry agent. Dead sessions return nulls without crashing the endpoint.

Follow the `useGsdRegistry.ts` polling pattern exactly:

- Export types: `AgentStateHint`, `PressureLevel`, `AgentLiveStatus` (interface with `state`, `contextPressure`, `contextPressureLevel` fields).
- Export `useAgentLiveStatus()` hook:
  - State: `Map<string, AgentLiveStatus>` (keyed by agentId).
  - `useEffect` with `setInterval` at `5_000ms`.
  - Fetch `GET /api/gsd/agents/live-status`.
  - Parse response as `{ agents: Array<{ agentId: string; state: AgentStateHint | null; contextPressure: number | null; contextPressureLevel: PressureLevel | null }> }`.
  - Build Map from agentId → `{ state, contextPressure, contextPressureLevel }`.
  - On fetch error → leave previous data in place (no setStatusMap call).
  - Cleanup: clear interval on unmount.
  - Return the Map directly.

**File 2: `src/client/hooks/useAgentStateFiles.ts`**

- Export types: `GsdStateInfo` (interface with `phase: string | null` and `progress: number | null`).
- Export `useAgentStateFiles(sessionNames: string[]): Map<string, GsdStateInfo>` hook:
  - State: `Map<string, GsdStateInfo>` (keyed by sessionName).
  - `useEffect` with dependency on `sessionNames.join(',')` (stable string key).
  - If `sessionNames.length === 0` → return early (no interval, no fetch).
  - Fetch all STATE.md endpoints in parallel via `Promise.allSettled`:
    - For each session: `GET /api/gsd/sessions/${session}/state`.
    - Parse response as `{ stateContent: string | null }`.
    - Parse stateContent with `parseStateFile()` (internal helper).
  - `parseStateFile(stateContent: string | null): GsdStateInfo`:
    - If null → `{ phase: null, progress: null }`.
    - Phase regex: `/^Phase:\s+(\d+)/m`.
    - Progress regex: `/Progress:.*?(\d+)%/m`.
  - `setInterval` at `30_000ms`.
  - Cleanup: clear interval on unmount.
  - Return the Map directly.
