  <action>
    Create `src/client/components/PluginRegistryView.tsx`:
    - Accept props: `plugins: Record<string, PluginModule>`, `enabledState: Record<string, boolean>`, `onToggle: (pluginId: string) => void`
    - Render a styled table (use `warden-*` Tailwind tokens throughout):
      - Header row: Name, Version, Description, Slot, Status
      - Table header classes: `text-warden-text-dim border-b border-warden-border`
      - Body rows: iterate `Object.values(plugins)`, display manifest fields
      - Status column: clickable toggle button that calls `onToggle(manifest.id)`
        - Enabled state: `bg-warden-success/20 text-warden-success` with text "Enabled"
        - Disabled state: `bg-warden-border/50 text-warden-text-dim` with text "Disabled"
      - Empty state: "No plugins registered" centered message when entries.length === 0
    - Wrap table in `<div className="flex-1 overflow-auto p-4">` with `<h2>Plugin Registry</h2>` heading

    Create `src/client/components/PluginSlotRenderer.tsx`:
    - Accept props: `slot: PluginSlot`, `enabledPlugins: PluginModule[]`
    - Filter `enabledPlugins` to only those matching the given `slot` via `plugin.manifest.slot === slot`
    - If no plugins match the slot, return `null` (render nothing)
    - For each matching plugin, render `<ErrorBoundary>` wrapper around `<PanelComponent />`
      - ErrorBoundary `key={manifest.id}`
      - ErrorBoundary `fallback` prop: a div with text `Plugin "{manifest.name}" failed to render.` styled with `text-warden-error`
    - Import `ErrorBoundary` from `./ErrorBoundary.js` and types from `@shared/pluginTypes.js`
  </action>
  <verify>
    Run `npx tsc --noEmit` — no type errors in either component.
  </verify>
  <done>

  </done>
