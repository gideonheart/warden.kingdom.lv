<objective>
Implement browser notification opt-in for permission prompts. A bell icon in the terminal header lets the operator toggle notifications. When enabled and the browser tab is unfocused, a desktop notification fires on state-transition (agent enters permission_prompt state). Clicking the notification focuses the tab and switches to the alerting session. Notification does not repeat while the agent stays in the same state.

Purpose: Operators often monitor multiple agents with Warden running in a background tab. Browser notifications alert them when an agent needs manual approval, reducing response time from minutes (next manual check) to seconds (notification appears).

Output: useBrowserNotifications hook, bell icon in TerminalView header, localStorage-persisted opt-in.
</objective>

**Step 1: Create useBrowserNotifications hook**

Create `src/client/hooks/useBrowserNotifications.ts`:

```typescript
import { useState, useEffect, useCallback, useRef } from 'react';
import type { AgentLiveStatus } from './useAgentLiveStatus.js';
import type { AgentInstance } from '@shared/types.js';
```

Constants:
```typescript
const NOTIFICATION_STORAGE_KEY = 'warden:notifications-enabled';
```

Interface:
```typescript
interface UseBrowserNotificationsParams {
  sessionStatusMap: Map<string, AgentLiveStatus>;
  instances: AgentInstance[];
  onSelectSession: (sessionName: string) => void;
}

interface UseBrowserNotificationsResult {
  notificationsEnabled: boolean;
  toggleNotifications: () => void;
  notificationPermission: NotificationPermission | 'unsupported';
}
```

Implementation details:

1. **Feature detection**: Guard against environments where `Notification` is undefined (e.g., SSR, older browsers). Return `notificationPermission: 'unsupported'` in that case.

2. **State initialization**: Read `localStorage` for the persisted toggle value:
   ```typescript
   const [notificationsEnabled, setNotificationsEnabled] = useState(() => {
     try {
       return localStorage.getItem(NOTIFICATION_STORAGE_KEY) === 'true';
     } catch {
       return false;
     }
   });
   ```

3. **Permission tracking**: Track `Notification.permission` in state so the UI can show different states:
   ```typescript
   const [permission, setPermission] = useState<NotificationPermission | 'unsupported'>(() => {
     if (typeof Notification === 'undefined') return 'unsupported';
     return Notification.permission;
   });
   ```

4. **State-transition tracking**: Use `useRef<Set<string>>` to track which `tmuxSessionName` values are currently in `permission_prompt` state. Only fire a notification when a session ENTERS the set (transition detection).
   ```typescript
   const permissionStateSessionsRef = useRef<Set<string>>(new Set());
   ```

5. **Main effect** — runs when `sessionStatusMap`, `instances`, or `notificationsEnabled` changes:
   ```typescript
   useEffect(() => {
     if (!notificationsEnabled || typeof Notification === 'undefined' || Notification.permission !== 'granted') {
       return;
     }

     const currentPermissionSessions = new Set<string>();

     for (const [sessionName, status] of sessionStatusMap) {
       if (status.state !== 'permission_prompt') continue;
       currentPermissionSessions.add(sessionName);

       // Only fire notification on state TRANSITION (new entry, not sustained)
       if (!permissionStateSessionsRef.current.has(sessionName)) {
         // Only fire when tab is NOT focused
         if (document.visibilityState === 'hidden') {
           const notification = new Notification('Warden — Permission Required', {
             body: `${sessionName} needs operator approval`,
             tag: `warden-permission-${sessionName}`,
           });
           notification.onclick = () => {
             window.focus(); // unreliable on macOS Chrome/Firefox, but attempt it
             onSelectSession(sessionName);
             notification.close();
           };
         }
       }
     }

     permissionStateSessionsRef.current = currentPermissionSessions;
   }, [sessionStatusMap, instances, notificationsEnabled, onSelectSession]);
   ```

6. **Toggle callback**:
   ```typescript
   const toggleNotifications = useCallback(() => {
     if (typeof Notification === 'undefined') return;

     if (!notificationsEnabled) {
       // Turning ON — check permission first
       if (Notification.permission === 'default') {
         // Must be triggered by user gesture (button click) — browser blocks otherwise
         Notification.requestPermission().then((result) => {
           setPermission(result);
           if (result === 'granted') {
             setNotificationsEnabled(true);
             try { localStorage.setItem(NOTIFICATION_STORAGE_KEY, 'true'); } catch {}
           }
           // If 'denied', do nothing — browser won't ask again
         });
       } else if (Notification.permission === 'granted') {
         setNotificationsEnabled(true);
         try { localStorage.setItem(NOTIFICATION_STORAGE_KEY, 'true'); } catch {}
       }
       // If 'denied', toggle does nothing — UI should show tooltip
     } else {
       // Turning OFF
       setNotificationsEnabled(false);
       try { localStorage.setItem(NOTIFICATION_STORAGE_KEY, 'false'); } catch {}
     }
   }, [notificationsEnabled]);
   ```

7. **Return value**:
   ```typescript
   return { notificationsEnabled, toggleNotifications, notificationPermission: permission };
   ```
