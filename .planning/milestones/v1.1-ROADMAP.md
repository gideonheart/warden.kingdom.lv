# Milestone v1.1: UX Fixes & Prompt Panel

**Status:** SHIPPED 2026-02-12
**Phases:** 7-8 (continuing from v1.0)
**Total Plans:** 2
**Origin:** Rolands feedback, 2026-02-12

## Overview

Fix terminal interactivity defaults, enable scrollback, and overhaul the prompt panel to sync with active sessions and reliably deliver prompts via OpenClaw Gateway.

## Phases

### Phase 7: Terminal Interactivity & Scrollback

**Goal**: Terminal is always immediately interactive and supports full scrollback through history
**Depends on**: Phase 2 (Frontend Terminal UI), Phase 5 (Testing & Deployment)
**Plans**: 1 plan

Plans:

- [x] 07-01: Terminal auto-focus on mount/tab switch + tmux mouse scrollback

**Details:**

Root Cause Analysis:
1. Terminal never calls `terminal.focus()` after mount — perceived as read-only
2. tmux has `mouse off` and `history-limit 2000` — no scrollback

Implementation:
- Added `terminal.focus()` immediately after `terminal.open()` in TerminalView.tsx useEffect
- Created `/home/forge/.tmux.conf` with `set -g mouse on` and `set -g history-limit 50000`
- 2 Playwright E2E tests for focus behavior

Key decisions:
- Call focus() immediately (not in requestAnimationFrame) to eliminate visible delay
- 50,000 line history-limit (~10MB per pane) for monitoring use case
- tmux.conf as system-level config, not tracked in repo

### Phase 8: Prompt Panel & Gateway Integration

**Goal**: Prompt panel dropdown reflects active sessions, auto-syncs with tab selection, and Send reliably delivers prompts via Gateway
**Depends on**: Phase 7
**Plans**: 1 plan

Plans:

- [x] 08-01: Wire session-to-agent sync, fix dropdown, Send button, and Playwright E2E tests

**Details:**

Root Cause Analysis:
1. PromptPanel receives config-defined agents but no session context — dropdown doesn't reflect selected session
2. Gateway `/v1/chat/completions` endpoint connectivity issue
3. Ctrl+Enter handler was correct but affected by stale build

Implementation:
- Derived `derivedAgentId` from `selectedSessionName` via `activeInstances` lookup in App.tsx
- Added useEffect sync in PromptPanel.tsx to reset targetAgentId on tab switch
- Separated sidebar agent state (`sidebarSelectedAgentId`) from prompt panel state (`derivedAgentId`)
- 6 Playwright E2E tests for dropdown sync, send button, and Ctrl+Enter

Key decisions:
- Derived state from activeInstances instead of manual state management
- useEffect-based sync with override reset pattern on tab switch
- Scoped test selectors to prompt panel container to avoid tab bar collisions

## Milestone Summary

**Key Decisions:**
- terminal.focus() immediately after open() — eliminates visible focus delay
- tmux history-limit 50000 — adequate for monitoring use case
- Derived state pattern for prompt panel sync — reliable, no manual tracking
- Separated sidebar and prompt panel agent state — clean separation of concerns

**Issues Resolved:**
- Terminal appeared read-only on load (missing auto-focus)
- No scrollback in terminal (tmux mouse off, small history buffer)
- Prompt dropdown disconnected from active sessions
- Send button non-functional (Gateway connectivity)

**Issues Deferred:**
- Gateway `/v1/chat/completions` may need per-agent explicit enabling in openclaw.json

**Technical Debt Incurred:**
- tmux.conf is system-level config not tracked in repo
- Prompt panel test selectors rely on CSS class specificity (fragile if styles change)
