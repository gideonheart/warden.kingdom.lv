2. Implement the class with these exact characteristics:

   ```typescript
   import { Bot, GrammyError, HttpError } from 'grammy';
   import { autoRetry } from '@grammyjs/auto-retry';
   ```

   **Class: TelegramBotService**
   - Private field: `bot: Bot | null = null`

   **Method: start(): void** (synchronous return — NOT async)
   - Read `process.env.WARDEN_TELEGRAM_BOT_TOKEN`
   - If no token: `console.warn('[TelegramBot] WARDEN_TELEGRAM_BOT_TOKEN not set — bot disabled')` and return
   - Create `new Bot(token)`
   - Call `this.bot.api.config.use(autoRetry())`
   - Install `this.bot.catch()` handler that logs errors with `[TelegramBot]` prefix:
     - GrammyError → log `err.description`
     - HttpError → log the error
     - Other → log as unexpected
   - Call `this.bot.start({ onStart: (botInfo) => console.log(\`[TelegramBot] Bot started: @\${botInfo.username}\`) })` with `.catch()` for fatal errors — do NOT await

   **Method: async stop(): Promise<void>**
   - If `this.bot` is null, return immediately
   - Try `await this.bot.stop()` with `console.log('[TelegramBot] Bot stopped cleanly')`
   - Catch errors and log them
   - Finally set `this.bot = null`

   **Method: isRunning(): boolean**
   - Return `this.bot?.isRunning() ?? false`

3. Export singleton: `export const telegramBotService = new TelegramBotService();`

4. CRITICAL security check: Verify that at no point does any console.log/warn/error call include the `token` variable or `process.env.WARDEN_TELEGRAM_BOT_TOKEN`. Only log the presence/absence indicator `[set]`/`[not set]`, or use `[TelegramBot]` prefix contextual messages.
