    **Step 2 — Add `sendToTopic()` to TelegramBotService:**

    Add the following method to the `TelegramBotService` class in `src/server/services/TelegramBotService.ts`, after the `isRunning()` method:

    ```typescript
    /**
     * Send a text message to a specific Telegram group topic.
     *
     * No-op when the bot is not running (token not configured or bot stopped).
     * Errors are caught and logged — never throws to callers.
     */
    async sendToTopic(chatId: string, topicId: string, text: string): Promise<void> {
      if (!this.bot) {
        console.warn('[TelegramBot] sendToTopic called but bot is not running — skipping');
        return;
      }
      try {
        await this.bot.api.sendMessage(chatId, text, {
          message_thread_id: parseInt(topicId, 10),
          parse_mode: 'Markdown',
        });
      } catch (error) {
        console.error(`[TelegramBot] Failed to send to topic ${topicId}:`, error);
      }
    }
    ```

    Key points:
    - `parseInt(topicId, 10)` converts string topicId from OpenClaw config to number for grammy
    - `parse_mode: 'Markdown'` enables triple-backtick code blocks in the excerpt
    - Errors caught and logged — never propagate to NotificationPoller (fire-and-forget semantics)
    - No-op guard when bot is null (token not configured)

    **Step 3 — Create NotificationPoller:**

    Create `src/server/services/NotificationPoller.ts`:

    ```typescript
    import { execFile } from 'child_process';
    import { promisify } from 'util';
    import stripAnsi from 'strip-ansi';
    import { instanceTracker } from './InstanceTracker.js';
    import { telegramBotService } from './TelegramBotService.js';
    import { openClawConfigReader } from './OpenClawConfigReader.js';
    import { detectAgentState } from '../utils/agentStateDetection.js';
    import { NotificationDeduplicator } from './NotificationDeduplicator.js';

    const execFileAsync = promisify(execFile);
    const POLL_INTERVAL_MS = 10_000;
    const PANE_LINES = 20;
    const EXCERPT_MAX_CHARS = 500;
    ```

    Class structure:
    - `private pollInterval: ReturnType<typeof setInterval> | null = null;`
    - `private deduplicator = new NotificationDeduplicator();`
    - `startPolling(): void` — sets interval, logs start message
    - `stopPolling(): void` — clears interval, sets to null
    - `private async pollAllSessions(): Promise<void>` — gets active instances from `instanceTracker.listActiveInstances()`, calls `Promise.allSettled(instances.map(i => this.pollSession(i.tmuxSessionName, i.agentId)))`
    - `private async pollSession(sessionName: string, agentId: string): Promise<void>`:
      1. `execFileAsync('tmux', ['capture-pane', '-pt', `${sessionName}:0.0`, '-S', `-${PANE_LINES}`])`
      2. Strip ANSI from pane output BEFORE calling `detectAgentState()` (Pitfall 1 from research — ANSI codes around cursor character break the regex)
      3. `const state = detectAgentState(cleanPane)`
      4. `const shouldNotify = this.deduplicator.recordAndCheck(sessionName, state)`
      5. If shouldNotify: extract excerpt (last EXCERPT_MAX_CHARS of cleanPane, trimmed), call `sendPermissionNotification()`
      6. Wrap entire body in try/catch — dead sessions silently ignored
    - `private async sendPermissionNotification(agentId: string, sessionName: string, excerpt: string): Promise<void>`:
      1. `const mappings = await openClawConfigReader.getTopicMappings()`
      2. `const mapping = mappings.find(m => m.agentId === agentId)`
      3. If no mapping: log warning and return
      4. Compose message text: header line with agent name and session, then triple-backtick code block with excerpt
      5. `await telegramBotService.sendToTopic(mapping.groupId, mapping.topicId, text)`
      6. Wrap in try/catch — log error but never throw

    Message format:
    ```
    ⚠️ *${agentId}* needs permission

    Session: \`${sessionName}\`
    ```
    ${excerpt}
    ```
    ```

    Export singleton: `export const notificationPoller = new NotificationPoller();`

    Important: The `startPolling()` method should also run one immediate poll (call `void this.pollAllSessions()` before the interval), matching the `InstanceTracker.startPeriodicSync()` pattern which calls `this.syncWithTmux()` immediately.
