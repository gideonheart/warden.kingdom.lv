  <action>
Make four changes to `src/client/components/TerminalView.tsx`:

**Change 1 — Add Enter to MOBILE_KEYS (MOB-01):**
Insert `{ label: 'Enter', seq: '\r' }` as the FIRST entry in the `MOBILE_KEYS` array (line 78). Use `\r` (carriage return), NOT `\n`. Position 0 ensures Enter is always visible without horizontal scrolling on narrow phones.

**Change 2 — Extend MobileKeyToolbarProps (MOB-02):**
Add `terminalRef: React.MutableRefObject<Terminal | null>` to the `MobileKeyToolbarProps` interface. Import `Terminal` type is already available from the existing `import { Terminal } from 'xterm'` at line 2.

**Change 3 — Add refocusTerminal helper and wire all onTouchStart handlers (MOB-02):**
Inside the `MobileKeyToolbar` function body, destructure `terminalRef` from props and create:
```typescript
const refocusTerminal = () => {
  terminalRef.current?.textarea?.focus();
};
```
Then add `refocusTerminal()` as the FIRST call after `event.preventDefault()` in EVERY `onTouchStart` handler. There are 5 handlers that need this:

1. **Copy/Close button** (line ~148): `onTouchStart={(event) => { event.preventDefault(); refocusTerminal(); onToggleCopyMode(); }}`
2. **Paste button** (line ~158): `onTouchStart={(event) => { event.preventDefault(); refocusTerminal(); handlePaste(); }}` — async `handlePaste()` is fine because focus is restored synchronously before the await boundary
3. **Esc button** (line ~164): `onTouchStart={(event) => { event.preventDefault(); refocusTerminal(); sendInput('\x1b'); }}`
4. **MOBILE_KEYS map** (line ~173): `onTouchStart={(event) => { event.preventDefault(); refocusTerminal(); sendInput(key.seq); }}`
5. **Cancel button in paste panel** (line ~138): `onTouchStart={(event) => { event.preventDefault(); refocusTerminal(); setShowPasteInput(false); }}`

IMPORTANT: Call `refocusTerminal()` synchronously — do NOT wrap in `requestAnimationFrame`, `setTimeout`, or any deferred call. iOS Safari ignores programmatic `.focus()` calls that are not synchronous within a user interaction handler.

IMPORTANT: Use `terminal.textarea?.focus()` (the DOM element), NOT `terminal.focus()` (the xterm.js public method). The public method does not reliably retain the iOS soft keyboard.

**Change 4 — Pass terminalInstanceRef to MobileKeyToolbar (MOB-02):**
In the `TerminalViewInner` JSX (line ~850), add the `terminalRef` prop:
```tsx
{IS_TOUCH_DEVICE && (
  <MobileKeyToolbar
    sendInput={sendInput}
    selectMode={selectMode}
    onToggleCopyMode={handleToggleCopyMode}
    terminalRef={terminalInstanceRef}
  />
)}
```

Do NOT modify `handleToggleCopyMode` — the existing `requestAnimationFrame` pattern there is desktop-correct and harmless; the new synchronous `refocusTerminal()` in the Copy button's `onTouchStart` handles the mobile case before `onToggleCopyMode` runs.
