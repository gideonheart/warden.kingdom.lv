## Overview

Browser-based terminal multiplexer for monitoring and intervening with OpenClaw autonomous agents. Live xterm.js streaming, multi-session tabs, agent metadata sidebar, prompt injection via Gateway API, session history, token usage dashboard, and production Nginx deployment.

## Milestone Summary

**Key Decisions:**
- Express 5 + Socket.IO 4 + React 19 + xterm.js 5 + node-pty + SQLite — confirmed working stack
- @tailwindcss/vite plugin required for Tailwind v4 (not just postcss/autoprefixer)
- SRP service architecture (TmuxSessionManager, TerminalStreamService, OpenClawConfigReader, etc.)
- Read-only mode removed — terminal always interactive (single operator model, read-only adds friction)
- Gateway API at localhost:3434 with bearer token auth
- SOUL.md preview truncated at last newline after 100 chars (max 200 chars)

**Issues Resolved:**
- FitAddon.fit() throws when container has zero dimensions — guarded with try/catch
- PTY resize ioctl EBADF crashes server when tmux session exits — guarded with isAlive check
- Terminal buffer clearing on mode toggle — fixed by removing mode system entirely
- User confusion about prompt destination — fixed with "via OpenClaw Gateway" labels

**Issues Deferred:**
- No unit tests — Playwright E2E provides integration coverage
- Token usage data population requires automated scraping
- ROADMAP.md progress table not auto-updated during execution

**Technical Debt Incurred:**
- PTY resize EBADF and FitAddon zero dimensions mitigated with guards, not fixed at source
- Token usage schema exists but no automated data population
