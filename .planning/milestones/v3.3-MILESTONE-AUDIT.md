### E2E Flows Verified

**Flow 1: Bot Startup (BOT-01, BOT-02, BOT-03, BOT-04)**
`index.ts` → constructs `ApprovalCallbackHandler` → `registerCallbackHandler(bot => handler.register(bot))` → `telegramBotService.start()` [reads token, applies autoRetry, installs callback handlers, starts long polling] → `notificationPoller.startPolling()` → `budgetAlertPoller.startPolling()` ✅

**Flow 2: Permission Prompt → Telegram Notification with Approve Button (PERM-01–05, APRV-01)**
`NotificationPoller.pollAllSessions()` → `database.getNotificationConfig()` → `permissionAlertsEnabled` check → `pruneExpired()` → `listActiveInstances()` → per session: `tmux capture-pane` → `stripAnsi()` → `detectAgentState()` → `deduplicator.recordAndCheck(sessionName, state, permissionCooldownMs)` → `sendPermissionNotification()` → `getTopicMappings()` → `sendToTopicWithApproveButton(groupId, topicId, text, sessionName)` → `approvalStateTracker.register(sessionName, {chatId, messageId, topicId, originalText})` ✅

**Flow 3: Operator Taps Approve Button (APRV-02–05)**
Telegram callback query → `bot.callbackQuery(/^approve:(.+)$/)` → `ApprovalCallbackHandler` → validate operator ID (APRV-03) → check expiry (APRV-05) → check consumed (double-tap) → `markConsumed()` → `sendPromptToSession(sessionName, '1')` (APRV-02) → `editMessageText(..., 'Approved at HH:MM')` without `reply_markup` (APRV-04) ✅

**Flow 4: Budget Alert Pipeline (BUDG-01, BUDG-02)**
`BudgetAlertPoller.pollBudgets()` → `database.getNotificationConfig()` → `budgetAlertsEnabled` check → `database.getBudgetAlertStatus()` → per agent: `checkAgent(status, budgetCooldownMs)` → threshold detection (amber 80% / red 100%) → deduplication + escalation check → `sendBudgetAlert()` → `getTopicMappings()` → `telegramBotService.sendToTopic(groupId, topicId, text)` ✅

**Flow 5: Settings Panel Round-Trip (NSET-01, NSET-02, NSET-03)**
`HistoryView` Notifications tab → `NotificationSettingsPanel.fetchConfig()` → `GET /api/notifications/config` → `database.getNotificationConfig()` + `telegramBotService.isRunning()` → renders bot status, toggles, cooldown inputs → user toggles/changes value → `saveField()` → `PUT /api/notifications/config` → `database.setNotificationConfig(patch)` → next poll cycle reads fresh config ✅

**Flow 6: Graceful Shutdown (BOT-03)**
`handleShutdown()` → `notificationPoller.stopPolling()` → `budgetAlertPoller.stopPolling()` → `await telegramBotService.stop()` → `httpServer.close()` → `database.close()` ✅
