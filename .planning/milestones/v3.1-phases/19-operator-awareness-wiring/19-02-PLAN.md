**Create `src/client/hooks/useGlobalHotkeys.ts`:**

Build a single-file hook, under 60 lines. Key aspects:

1. **Focus guard function** (`isInTextInput`):
   ```typescript
   function isInTextInput(target: EventTarget | null): boolean {
     if (!target) return false;
     const element = target as HTMLElement;
     return element.tagName === 'INPUT' || element.tagName === 'TEXTAREA' || element.isContentEditable;
   }
   ```

2. **Hook interface:**
   ```typescript
   import { useEffect } from 'react';
   import type { AgentInstance } from '@shared/types.js';

   interface UseGlobalHotkeysParams {
     instances: AgentInstance[];
     selectedSessionName: string | null;
     onSelectSession: (sessionName: string) => void;
     onToggleSidebar: () => void;
     terminalFocusRef: React.MutableRefObject<(() => void) | null>;
     currentView: string;
   }
   ```

3. **Single keydown handler registered with `{ capture: true }`:**
   - This is CRITICAL — capture phase fires before xterm.js bubble-phase listeners, preventing shortcut keys from being forwarded to the PTY.
   - Call `event.preventDefault()` and `event.stopPropagation()` for all handled shortcuts.

4. **Handler logic (merged Ctrl+key and Escape into one handler):**

   First: if `isInTextInput(event.target)`, return immediately (KB-05 focus guard).

   **Escape key** (no ctrlKey required):
   - Only in terminals view.
   - Check if the terminal already has focus by testing `!document.activeElement?.closest('.xterm')`. If terminal does NOT have focus, call `terminalFocusRef.current?.()` to refocus. Do NOT call `event.preventDefault()` — Escape must still reach the PTY when terminal already has focus. Do NOT call `event.stopPropagation()` — this allows Escape to work naturally in xterm TUIs.

   **Ctrl+F** (any view):
   - `event.preventDefault()` + `event.stopPropagation()` — prevent browser find bar.
   - Return. This is a stub for Phase 20.

   **Ctrl+B** (any view):
   - `event.preventDefault()` + `event.stopPropagation()`.
   - Call `onToggleSidebar()`.

   **Ctrl+1 through Ctrl+9** (terminals view only):
   - Parse `event.key` as integer. If 1-9 and `instances[digit-1]` exists, call `onSelectSession(instances[digit-1].tmuxSessionName)`.
   - `event.preventDefault()` + `event.stopPropagation()`.
   - Out-of-range silently no-ops.

   **Ctrl+[** (terminals view only):
   - Find current index in instances. Wrap: if index is 0 or -1, go to last instance.
   - `event.preventDefault()` + `event.stopPropagation()`.
   - Also check `event.code === 'BracketLeft'` as fallback for non-QWERTY layouts.

   **Ctrl+]** (terminals view only):
   - Find current index. Wrap: if at last, go to first.
   - `event.preventDefault()` + `event.stopPropagation()`.
   - Also check `event.code === 'BracketRight'` as fallback.

5. **Registration:**
   ```typescript
   document.addEventListener('keydown', handleKeyDown, { capture: true });
   return () => {
     document.removeEventListener('keydown', handleKeyDown, { capture: true });
   };
   ```

6. **Dependency array:**

   ```typescript
   [instances, selectedSessionName, onSelectSession, onToggleSidebar, terminalFocusRef, currentView]
   ```

  <files>src/client/components/TerminalView.tsx</files>
  <action>
Inside `TerminalView.tsx`, in the terminal initialization `useEffect` (the one that creates `new Terminal({...})`), immediately after `terminal.focus()` and before the `fitAddon.fit()` call, add `attachCustomKeyEventHandler`:

```typescript
// Suppress shortcut keys from being forwarded to the PTY.
// The global useGlobalHotkeys handler (capture phase) handles these keys
// at the document level. Without this guard, xterm.js also processes the
// keydown event and sends escape sequences (e.g. ^1, ^[, ^]) to the PTY.
terminal.attachCustomKeyEventHandler((event: KeyboardEvent) => {
  // Ctrl+F: suppress browser find AND PTY forwarding
  if (event.ctrlKey && (event.key === 'f' || event.key === 'F')) {
    return false;
  }
  // Ctrl+B: suppress PTY forwarding (Ctrl+B is tmux prefix by default in some configs)
  if (event.ctrlKey && (event.key === 'b' || event.key === 'B')) {
    return false;
  }
  // Ctrl+1..9: suppress PTY forwarding
  if (event.ctrlKey && /^[1-9]$/.test(event.key)) {
    return false;
  }
  // Ctrl+[ and Ctrl+]: suppress PTY forwarding
  if (event.ctrlKey && (event.key === '[' || event.key === ']')) {
    return false;
  }
  return true;
});
```

**Reasoning for each suppression:**
- `Ctrl+F`: Per STATE.md: "attachCustomKeyEventHandler for Ctrl+F requires both event.preventDefault() AND return false — missing either opens browser native find bar or injects to PTY"
- `Ctrl+B`: Would otherwise send `^B` (tmux prefix key) to the PTY, potentially activating tmux commands
- `Ctrl+1-9`: Would send `^1` through `^9` escape sequences to the PTY
- `Ctrl+[/]`: Would send `^[` (ESC) or `^]` to the PTY

Do NOT suppress Escape in attachCustomKeyEventHandler — Escape is intentionally let through to xterm so it works in TUI applications (vim, etc.). The global handler only refocuses the terminal when it does NOT already have focus.

Run the full build after this change to ensure the production build succeeds.
