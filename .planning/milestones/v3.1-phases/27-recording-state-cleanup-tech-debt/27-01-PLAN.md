The problem: when a PTY exits, TerminalStreamService emits `terminal:exit` to all subscribers. `useTerminalSocket` calls `onSessionExit` (→ `handleSessionExit` in TerminalView). However `useRecordingState` has no knowledge of the session exit event, so `isRecording` stays `true` and the REC button remains red indefinitely.

The fix — expose `sessionExited` from `useRecordingState` and call it from `handleSessionExit` in TerminalView:

**In `src/client/hooks/useRecordingState.ts`:**

1. Add `sessionExited` to the `UseRecordingStateResult` interface:
   ```typescript
   sessionExited: () => void;
   ```

2. Implement `sessionExited` as a `useCallback` inside the hook body. It must:
   - Clear the elapsed ticker: `if (tickerRef.current) { clearInterval(tickerRef.current); tickerRef.current = null; }`
   - Reset `startedAtRef.current = null`
   - Reset state: `setIsRecording(false)`, `setElapsedMs(0)`, `setRecordingId(null)`
   - Call `onRecordingStoppedRef.current?.('session_ended')` so the parent (TerminalView) gets notified of the reason
   - Do NOT make any HTTP call to the stop endpoint — the server already auto-stopped the recording via `RecordingCaptureService.stopRecording(..., 'session_ended')` in `TerminalStreamService.ptyProcess.onExit`

   ```typescript
   const sessionExited = useCallback(() => {
     if (tickerRef.current) {
       clearInterval(tickerRef.current);
       tickerRef.current = null;
     }
     startedAtRef.current = null;
     setIsRecording(false);
     setElapsedMs(0);
     setRecordingId(null);
     onRecordingStoppedRef.current?.('session_ended');
   }, []);
   ```

3. Add `sessionExited` to the return object.

**In `src/client/components/TerminalView.tsx`:**

1. Destructure `sessionExited` from `useRecordingState(...)`.

2. Modify `handleSessionExit` to also call `sessionExited()`:
   ```typescript
   const handleSessionExit = useCallback((exitCode: number) => {
     sessionExited();
     onSessionExit(tmuxSessionName, exitCode);
   }, [tmuxSessionName, onSessionExit, sessionExited]);
   ```

   `sessionExited` is safe to include in the dependency array — it is defined with `useCallback([])` (empty deps), so its identity is stable across renders.

Do NOT modify `useTerminalSocket.ts` — the socket event is already routed correctly through `onSessionExitRef.current`. Do NOT add any new socket import to `useRecordingState.ts`.
