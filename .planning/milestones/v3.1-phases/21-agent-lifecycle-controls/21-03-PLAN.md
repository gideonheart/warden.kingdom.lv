<objective>
Fix the gap where stopped and errored sessions disappear from the API response, preventing operators from clicking Restart on stopped tabs.

Purpose: `listActiveInstances()` currently filters to `status IN ('active', 'idle', 'starting', 'stopping')`, which excludes `'stopped'` and `'error'` sessions. Once a session stops, the next poll removes it from the tab bar. The client-side code in App.tsx already includes `'stopped'` and `'error'` in its `activeInstances` filter, and the Restart/dismiss buttons are already implemented in InstanceTabBar and TerminalView overlays. The only gap is the server-side query that drops these sessions before they reach the client.

Output: Updated `listActiveInstances()` SQL query that includes recently-stopped and recently-errored sessions (within the last 30 minutes based on `last_active_at` timestamp).
</objective>

  <action>
Update the `listActiveInstances()` method in `DatabaseConnection` to include `'stopped'` and `'error'` sessions that were updated within the last 30 minutes.

**Current query (line 71):**
```sql
WHERE status IN ('active', 'idle', 'starting', 'stopping')
```

**New query:**
```sql
WHERE status IN ('active', 'idle', 'starting', 'stopping')
   OR (status IN ('stopped', 'error') AND last_active_at >= datetime('now', '-30 minutes'))
```

This keeps the existing behavior for all non-terminal states untouched. For stopped/error sessions, it only returns those with `last_active_at` within the last 30 minutes — this ensures recently-stopped tabs remain visible for restart access while old stopped sessions eventually age out to prevent clutter.

**Why this approach works end-to-end:**
1. `listActiveInstances()` now returns recently-stopped sessions to the API caller
2. `GET /api/instances` calls `syncWithTmux()` which returns `listActiveInstances()` — no route changes needed
3. App.tsx `activeInstances` filter already includes `'stopped'` and `'error'` statuses (lines 57-64) — no client changes needed
4. InstanceTabBar already renders Restart/Dismiss buttons for stopped tabs — no component changes needed
5. TerminalView already has stopped/error overlays with Restart buttons — no component changes needed

**What NOT to change:**
- Do NOT modify `listAllInstances()` — it already returns everything unfiltered
- Do NOT modify `markMissingSessionsStopped()` — its `'active'/'idle'` guard is correct
- Do NOT modify `findActiveInstanceByAgentId()` — its duplicate check correctly excludes stopped sessions
- Do NOT modify any client-side code — the client already handles all 6 statuses

Add a comment above the new clause explaining the 30-minute retention window for stopped/error sessions and its purpose (restart access).
