**2. Burn Rate Fetch (on window change):**

Add state for burn rate data: `const [burnRates, setBurnRates] = useState<BurnRateEntry[]>([]);`

Add a useEffect that fetches `GET /api/history/burn-rate?window=${burnWindow}` whenever `burnWindow` changes. Parse response and set `burnRates`.

**3. Burn Rate Display (section below window selector):**

Per CONTEXT.md: "Show per-agent burn rates AND an aggregate total across all agents."

Create a "Burn Rate" section heading. Render cards in a grid layout (same grid as existing summary cards: `grid-cols-1 sm:grid-cols-2 lg:grid-cols-3`).

For each agent in `burnRates`, show a card with:
- Agent name (using the existing `formatAgentId()` helper)
- Burn rate: `$X.XXXX/hr` (4 decimal places for hourly rate)
- Window cost total: `$X.XX`

After per-agent cards, render an aggregate "All Agents" card showing:
- Total burn rate = sum of all agents' burnRatePerHour
- Total window cost = sum of all agents' windowCostUsd

**4. Budget Threshold Editor (integrated into burn rate cards):**

Per CONTEXT.md: "Inline number input per agent — edit in place, save on blur or Enter."

Add state for budget configs: `const [budgetConfigs, setBudgetConfigs] = useState<BudgetConfig[]>([]);`
Add state for budget alert statuses: `const [budgetStatuses, setBudgetStatuses] = useState<BudgetAlertStatus[]>([]);`
Add state for editing: `const [editingBudget, setEditingBudget] = useState<Record<string, string>>({});`

Fetch budget configs on mount: `GET /api/history/budget-config` -> set `budgetConfigs`.
Fetch budget statuses on mount and after save: `GET /api/history/budget-config/status` -> set `budgetStatuses`.

In each per-agent burn rate card, add below the rate display:
- A label "Daily budget:" with an inline `<input type="number" min="0" step="1">` showing the current budget (or empty placeholder if none set)
- On blur or Enter key: PUT `/api/history/budget-config/${agentId}` with `{ dailyBudgetUsd: value }`, then refetch budget configs and statuses
- Per CONTEXT.md: "No default budget — agents start with no budget ($0 / empty = no budget configured)"

**5. Budget Progress Bar:**

Per CONTEXT.md: "Budget progress bar per agent: green -> amber at 80% -> red at 100%."

For agents that have a budget configured (`dailyBudgetUsd > 0`), render a progress bar below the budget input:
- Width = `min(budgetPct, 100)%`
- Color: `bg-green-500` if < 80%, `bg-amber-500` if 80-99%, `bg-red-500` if >= 100%
- Show `budgetPct.toFixed(1)%` label and `$todayCostUsd / $dailyBudgetUsd` text

Agents without a budget show no progress bar (per CONTEXT.md: "Unconfigured agents never trigger alerts").

**6. Cost Projection Card:**

Per CONTEXT.md: "Projections show daily + weekly estimated spend at current burn rate."

Below the burn rate cards, add a projection card with `bg-warden-border/20 rounded-lg p-3`:
- Title: "Cost Projection ({window label})"
- Two columns: Est. Daily (`projectedDailyUsd` summed across agents, `$X.XX`), Est. Weekly (`projectedWeeklyUsd` summed, `$X.XX`)
- Subtitle: "at $X.XXXX/hr total"
- This recalculates automatically when window changes (it uses the same `burnRates` state)

**Styling:** Use existing warden theme tokens (`bg-warden-border/20`, `text-warden-accent`, `text-warden-text-dim`, `font-mono`). Match the visual density and spacing of the existing Per-Agent Summary cards.

**Important:** Do NOT remove any existing UI (summary cards, daily breakdown table, scan button, agent filter). The new sections are added ABOVE the existing "Per-Agent Summary" section.
