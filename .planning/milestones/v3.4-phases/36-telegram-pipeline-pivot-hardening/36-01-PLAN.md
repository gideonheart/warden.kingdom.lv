    **Step 1 — Add botToken to OpenClawConfig type:**
    In `src/shared/openclawTypes.ts`, update the `channels.telegram` type to include `botToken?: string` field. The type currently has `enabled`, `groups` — add `botToken` alongside them.

    **Step 2 — Add getBotToken() to OpenClawConfigReader:**
    In `src/server/services/OpenClawConfigReader.ts`, add a new method:
    ```typescript
    async getBotToken(): Promise<string | null> {
      const config = await this.getConfig();
      return config.channels?.telegram?.botToken ?? null;
    }
    ```

    **Step 3 — Rewrite TelegramBotService to send-only:**
    Completely rewrite `src/server/services/TelegramBotService.ts`:

    - Remove `grammy` Bot class instantiation and long-polling (`bot.start()`, `bot.stop()`).
    - Remove `registerCallbackHandler()`, `callbackHandlers` array, `isRunning()` method, and `sendToTopicWithApproveButton()` method.
    - Keep the class as a singleton, but change initialization:
      - Add `private botToken: string | null = null` and `private initialized = false`.
      - Add an `async initialize(): Promise<void>` method that calls `openClawConfigReader.getBotToken()` and stores the token. Log `[TelegramBot] Bot token loaded from openclaw.json — send-only mode` on success, or `[TelegramBot] No bot token in openclaw.json — notifications disabled` if null.
      - `sendToTopic(chatId, topicId, text)` becomes async and uses the raw Telegram Bot API via `fetch`:
        ```typescript
        const url = `https://api.telegram.org/bot${this.botToken}/sendMessage`;
        const response = await fetch(url, {
          method: 'POST',
          headers: { 'Content-Type': 'application/json' },
          body: JSON.stringify({
            chat_id: chatId,
            message_thread_id: parsedTopicId,
            text,
            parse_mode: 'Markdown',
          }),
        });
        ```
      - **FIX-04 — topicId validation:** At the top of `sendToTopic`, validate topicId:
        ```typescript
        const parsedTopicId = parseInt(topicId, 10);
        if (!Number.isFinite(parsedTopicId)) {
          console.warn(`[TelegramBot] Invalid topicId "${topicId}" — must be a finite integer. Skipping send.`);
          return;
        }
        ```
      - Guard: if `this.botToken` is null, log warning and return early (same pattern as before but checking botToken instead of bot instance).
      - Add `isConfigured(): boolean` method that returns `this.botToken !== null` (replaces `isRunning()` for the notification settings panel status).
    - Remove `grammy` and `@grammyjs/auto-retry` imports entirely from TelegramBotService.

      - Replace `telegramBotService.sendToTopicWithApproveButton(...)` with `telegramBotService.sendToTopic(...)`.
      - Remove the `if (messageId !== null)` block that registers with approvalStateTracker.
      - **FIX-03 — Escape Markdown in pane excerpt:** Before building the notification text, escape Markdown special characters in the `excerpt` string. Telegram Markdown v1 special chars are: `_`, `*`, `` ` ``, `[`. Create a helper function `escapeTelegramMarkdown(text: string): string` that replaces these with backslash-escaped versions: `text.replace(/([_*`\[])/g, '\\$1')`. Apply it to the excerpt BEFORE wrapping it in the triple-backtick code block. Note: since the excerpt is inside a code block (``` ``` ```), actually the safest approach is to strip backticks from the excerpt to avoid premature code block termination: `excerpt.replace(/`/g, "'")`. Apply this sanitization in the excerpt extraction.
