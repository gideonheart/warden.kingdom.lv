**Types (src/shared/types.ts):**
- Add `CrashRestartMode` type: `'none' | 'once' | 'always'`
- Add `RestartPolicy` interface: `{ agentId: string; crashRestartMode: CrashRestartMode; stormDisabledAt: string | null }`
- Add `LifecycleEventType` update: add `'auto-restarted'` to the existing union type (Plan 02 will use this)

**Database (src/server/database/DatabaseConnection.ts):**
- Add migration for `session_lifecycle_policy` table in `runMigrations()`:
  ```sql
  CREATE TABLE IF NOT EXISTS session_lifecycle_policy (
    agent_id TEXT PRIMARY KEY,
    crash_restart_mode TEXT NOT NULL DEFAULT 'none',
    storm_disabled_at TEXT,
    updated_at TEXT NOT NULL DEFAULT CURRENT_TIMESTAMP
  )
  ```
- Add `getRestartPolicy(agentId: string): RestartPolicy` method — returns policy for agent, defaulting to `{ agentId, crashRestartMode: 'none', stormDisabledAt: null }` if no row exists
- Add `setRestartPolicy(agentId: string, mode: CrashRestartMode): void` method — upserts the policy row, clears `storm_disabled_at` when operator explicitly changes the mode (since manual change means operator is aware)
- Add `getAllRestartPolicies(): RestartPolicy[]` method — returns all rows for the GET all endpoint
- Import the new types from shared/types.ts

**API (src/server/routes/instanceRoutes.ts):**
- Add `GET /api/restart-policies` — returns `{ policies: RestartPolicy[] }` from `database.getAllRestartPolicies()`
- Add `PUT /api/restart-policies/:agentId` — accepts `{ crashRestartMode: CrashRestartMode }` body, validates mode is one of 'none'/'once'/'always', calls `database.setRestartPolicy()`, returns `{ policy: RestartPolicy }`
- Import RestartPolicy and CrashRestartMode types
